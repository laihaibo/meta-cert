# 第三章：信息安全技术

> 分值占比：8%-12% | 重要程度：★★★★

## 考情分析

信息安全技术是系统架构设计师必须掌握的核心领域之一。本章涵盖信息安全基础、加密技术、密钥管理、访问控制、数字签名、抗攻击技术及安全评估等内容。

---

## 3.1 信息安全基础

### 3.1.1 信息安全目标（CIA三元组）

- **机密性（Confidentiality）**：信息不被未授权访问
- **完整性（Integrity）**：信息不被未授权篡改
- **可用性（Availability）**：授权用户能及时访问信息

### 3.1.2 信息安全威胁

- 物理安全威胁：火灾、水灾、地震
- 网络安全威胁：窃听、篡改、重放、拒绝服务
- 主机安全威胁：病毒、木马、漏洞利用
- 应用安全威胁：SQL注入、跨站脚本（XSS）

### 3.1.3 信息安全策略

- 技术措施：加密、防火墙、入侵检测
- 管理措施：安全制度、人员培训
- 法律措施：法律法规、标准规范

## 3.2 信息加解密技术

### 3.2.1 对称加密

- **特点**：加密密钥与解密密钥相同
- **代表算法**：DES、3DES、AES、SM4
- **优点**：加密速度快，适合大量数据加密
- **缺点**：密钥分发困难，不适合开放网络

### 3.2.2 非对称加密（公钥加密）

- **特点**：加密密钥与解密密钥不同（公钥公开，私钥保密）
- **代表算法**：RSA、ECC、SM2
- **优点**：密钥分发方便，支持数字签名
- **缺点**：加密速度慢，不适合大量数据加密

### 3.2.3 混合加密

- 使用非对称加密安全传输对称密钥
- 使用对称加密加密实际数据
- 结合两者优点

## 3.3 密钥管理技术

### 3.3.1 密钥生命周期

- 密钥生成 → 密钥分发 → 密钥存储 → 密钥使用 → 密钥撤销 → 密钥销毁

### 3.3.2 密钥分发技术

- **KDC（密钥分发中心）**：集中式密钥分发
- **Kerberos**：基于票据的认证协议
- **数字证书**：由CA签发的公钥证书

### 3.3.3 数字证书

- **X.509证书**：标准数字证书格式
- **CA（证书颁发机构）**：可信的第三方机构
- **证书链**：根CA → 中间CA → 用户证书

## 3.4 访问控制与数字签名

### 3.4.1 访问控制

**自主访问控制（DAC）**
- 资源所有者自主设置访问权限
- 基于访问控制列表（ACL）
- 灵活但安全性较低

**强制访问控制（MAC）**
- 系统统一管理访问权限
- 基于安全级别（绝密、机密、秘密、公开）
- 安全性高但灵活性差

**基于角色的访问控制（RBAC）**
- 权限赋予角色，角色赋予用户
- 用户与权限解耦
- 管理方便，适合企业应用

### 3.4.2 数字签名

**数字签名原理：**
1. 发送方使用私钥对消息摘要加密（签名）
2. 接收方使用发送方公钥验证签名
3. 提供身份认证、完整性、不可否认性

**常用算法：**
- RSA签名：基于RSA非对称加密
- DSA签名：基于离散对数问题
- SM2：国密签名算法

## 3.5 信息安全抗攻击技术

### 3.5.1 网络防火墙

- **包过滤防火墙**：基于IP、端口过滤
- **代理防火墙**：应用层代理，深度检测
- **状态检测防火墙**：跟踪连接状态
- **下一代防火墙（NGFW）**：集成入侵防御、应用识别

### 3.5.2 入侵检测与防御

- **IDS（入侵检测系统）**：检测攻击行为
- **IPS（入侵防御系统）**：检测并主动阻断攻击
- **检测技术**：特征检测、异常检测

### 3.5.3 VPN技术

- **IPsec VPN**：网络层加密
- **SSL VPN**：应用层加密
- **MPLS VPN**：运营商提供的虚拟专用网

### 3.5.4 恶意代码防护

- **病毒**：自我复制的恶意程序
- **木马**：伪装正常程序，开后门
- **蠕虫**：利用网络漏洞自动传播
- **勒索软件**：加密文件索要赎金

## 3.6 信息安全保障体系与评估

### 3.6.1 信息安全保障体系

- **技术体系**：防火墙、IDS、加密、认证
- **管理体系**：安全策略、风险评估、应急响应
- **运营体系**：安全监控、漏洞管理、事件处置

### 3.6.2 信息安全等级保护

- **第一级**：自主保护级
- **第二级**：指导保护级
- **第三级**：监督保护级（重要信息系统）
- **第四级**：强制保护级
- **第五级**：专控保护级

### 3.6.3 信息安全评估标准

- **TCSEC（橙皮书）**：美国可信计算机系统评估标准
- **ITSEC**：欧洲信息技术安全评估标准
- **CC（通用准则）**：国际通用安全评估标准

::: warning 重点难点
- **对称加密**：加密密钥 = 解密密钥（DES、AES、SM4）；速度快，适合大数据量
- **非对称加密**：加密密钥 ≠ 解密密钥（RSA、ECC、SM2）；速度慢，适合密钥交换
- **混合加密**：非对称加密解决密钥分发问题，对称加密解决效率问题
- **数字签名与加密的区别**：签名用私钥加密摘要，加密用公钥加密数据
- **RBAC核心思想**：权限与用户解耦，通过角色作为中间层
:::

## 1.1 计算机硬件基础

### 1.1.1 CPU与运算器

- **CPU主要组成**：运算器、控制器、寄存器组、内部总线
- **运算器功能**：执行算术运算（加、减、乘、除）和逻辑运算（与、或、非、异或）
- **控制器功能**：产生控制信号，协调各部件工作

### 1.1.2 存储器系统

**层次结构：**

```
寄存器 → Cache → 主存 → 辅存
   ↑        ↑        ↑        ↑
 最快    快      中等     慢
 最小    小       大      极大
 最贵    贵       便宜    便宜
```

**Cache工作原理（局部性原理）：**
- 时间局部性：最近访问的内容很可能再次被访问
- 空间局部性：当前访问位置附近的存储单元很可能被访问

### 1.1.3 I/O系统

- **输入输出方式**：程序查询、程序中断、DMA、通道方式
- **总线分类**：数据总线（DB）、地址总线（AB）、控制总线（CB）

## 1.2 计算机软件

### 1.2.1 系统软件

- **操作系统**：管理硬件与软件资源的核心系统软件
- **语言处理程序**：编译器、解释器、汇编程序
- **数据库管理系统（DBMS）**：管理数据库的软件系统
- **系统实用工具**：诊断程序、调试工具、性能监控工具

### 1.2.2 应用软件

- **通用应用软件**：办公软件、图形图像软件、浏览器等
- **专用应用软件**：行业定制软件、业务系统

---

## 1.3 嵌入式系统

### 1.3.1 嵌入式系统定义

嵌入式系统是嵌入到设备内部，为特定应用而设计的专用计算机系统。

### 1.3.2 嵌入式系统特点

- 专用性强，通常面向特定应用
- 系统精简，资源受限（CPU、内存、功耗）
- 软件固化在ROM/Flash中
- 实时性要求高（硬实时/软实时）

### 1.3.3 嵌入式操作系统

- **μC/OS-II**：开源、实时、可裁剪
- **VxWorks**：商业实时操作系统
- **Linux**：开源、可裁剪，广泛用于嵌入式设备
- **Android**：基于Linux的移动操作系统

## 1.4 计算机网络基础

### 1.4.1 OSI七层模型

| 层次 | 名称 | 主要功能 | 典型协议 |
|------|------|----------|----------|
| 7 | 应用层 | 提供应用程序接口 | HTTP、FTP、SMTP、DNS |
| 6 | 表示层 | 数据格式转换、加密解密 | SSL/TLS、JPEG、MPEG |
| 5 | 会话层 | 建立、管理和终止会话 | NetBIOS、RPC |
| 4 | 传输层 | 端到端数据传输 | TCP、UDP |
| 3 | 网络层 | 路由与寻址 | IP、ICMP、OSPF |
| 2 | 数据链路层 | 帧传输与差错控制 | Ethernet、PPP |
| 1 | 物理层 | 比特流传输 | 光纤、双绞线、无线 |

### 1.4.2 TCP/IP协议族

- **TCP**：面向连接、可靠传输（三次握手、确认重传）
- **UDP**：无连接、不可靠但高效
- **IP**：网络层协议，负责主机寻址与路由

### 1.4.3 网络设备

- 集线器（Hub）：物理层，广播转发
- 交换机（Switch）：数据链路层，基于MAC地址转发
- 路由器（Router）：网络层，基于IP地址路由

## 1.5 计算机语言基础

### 1.5.1 程序设计语言分类

- **低级语言**：机器语言、汇编语言
- **高级语言**：面向过程（C）、面向对象（C++、Java）、函数式（Haskell）

### 1.5.2 语言处理程序

- **汇编程序**：将汇编语言翻译为机器语言
- **编译程序**：将高级语言翻译为机器语言（一次性翻译）
- **解释程序**：边解释边执行，不生成目标程序

### 1.5.3 程序翻译过程

```
源程序 → 预处理 → 编译 → 汇编 → 链接 → 可执行程序
```

## 1.6 多媒体技术

### 1.6.1 多媒体基本概念

多媒体是指将文本、图形、图像、声音、动画、视频等多种媒体形式集于一体的技术。

### 1.6.2 多媒体数据特征

- **多样性**：多种信息媒体的集成
- **集成性**：多种媒体的有机集成
- **交互性**：人机双向信息交流
- **实时性**：声音、视频需实时处理

### 1.6.3 数据压缩

- **无损压缩**：JPEG-LS、PNG、GIF
- **有损压缩**：JPEG、MPEG、MP3

## 1.7 系统性能评价

### 1.7.1 性能指标

- **响应时间**：从请求到响应的时间
- **吞吐量**：单位时间内完成的请求数
- **并发用户数**：系统同时支持的用户数量
- **资源利用率**：CPU、内存、I/O利用率

### 1.7.2 性能测试方法

- **基准测试**：使用标准化测试程序
- **压力测试**：测试系统极限负载
- **负载测试**：测试不同负载下的性能表现

### 1.7.3 系统可靠性

- **可靠性**：系统在规定条件下无故障运行的能力
- **可用性**：系统可正常使用的时间比例
- **可维护性**：系统故障后恢复的速度

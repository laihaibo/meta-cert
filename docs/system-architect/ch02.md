# 第二章：信息系统基础知识

> 分值占比：8%-12% | 重要程度：★★★★

## 考情分析

本章介绍信息系统的基本概念、分类及典型架构模型。系统架构设计师需要掌握各类信息系统的功能定位、适用场景以及系统架构设计的基本原理。

---

## 2.1 信息系统概述

### 2.1.1 信息系统定义

信息系统是由人、硬件、软件、数据、过程等要素组成的，用于收集、处理、存储和分发信息的系统。

### 2.1.2 信息系统功能

- **数据采集**：从外部环境或业务过程中获取数据
- **数据加工**：对原始数据进行转换、计算、分析
- **数据存储**：将处理后的数据存储到数据库或文件中
- **数据传输**：在系统内部或系统之间传递数据
- **信息输出**：以报表、图形、查询等形式输出信息

### 2.1.3 信息系统分类

按管理层次：
- **战略层**：支持高层管理决策
- **管理层**：支持中层管理控制
- **操作层**：支持基层日常操作

---

## 2.2 业务处理系统（TPS）

### 2.2.1 TPS定义

事务处理系统（Transaction Processing System）是信息系统中最基础、最底层的系统，负责处理组织日常的、常规的事务。

### 2.2.2 TPS主要功能

- 记录日常业务数据
- 处理特定业务事务
- 生成业务报表

### 2.2.3 TPS特点

- 处理高度结构化的任务
- 数据量大、操作频繁
- 要求高可靠性和数据一致性

---

## 2.3 管理信息系统（MIS）

### 2.3.1 MIS定义

管理信息系统（Management Information System）是为管理决策提供信息支持的计算机系统。

### 2.3.2 MIS主要功能

- 提供结构化报告
- 支持中层管理决策
- 监控业务运行状态

### 2.3.3 MIS特点

- 面向结构化决策问题
- 提供定期报告
- 基于TPS数据生成管理报表

---

## 2.4 决策支持系统（DSS）

### 2.4.1 DSS定义

决策支持系统（Decision Support System）是为管理者提供半结构化或非结构化决策问题求解支持的交互式计算机系统。

### 2.4.2 DSS主要功能

- 支持半结构化、非结构化决策
- 集成数据、模型、知识
- 提供交互式分析工具

### 2.4.3 DSS组成

- **数据管理子系统**：内部数据、外部数据
- **模型管理子系统**：数学模型、分析模型
- **知识管理子系统**：专家知识、规则库
- **用户接口子系统**：人机交互界面

### 2.4.4 智能决策支持系统（IDSS）

在DSS基础上引入人工智能技术，能够提供更高级的决策支持能力。

---

## 2.5 专家系统（ES）

### 2.5.1 ES定义

专家系统（Expert System）是模拟人类专家决策能力的计算机程序系统。

### 2.5.2 ES主要组成

- **知识库**：存储专家知识
- **推理机**：运用知识进行推理
- **解释器**：解释推理过程
- **知识获取器**：获取新知识
- **人机接口**：用户交互

### 2.5.3 ES特点

- 模拟专家决策
- 基于规则或案例推理
- 可解释推理过程
- 知识可维护和更新

---

## 2.6 办公自动化系统（OAS）

### 2.6.1 OAS定义

办公自动化系统（Office Automation System）是利用计算机技术、网络技术和通信技术，提高办公效率和办公质量的系统。

### 2.6.2 OAS主要功能

- 文字处理：文档编辑、排版
- 电子表格：数据计算与分析
- 电子邮件：内外通信
- 日程管理：时间安排与提醒
- 文档管理：文档存储、检索与共享

---

## 2.7 典型信息系统架构模型

### 2.7.1 C/S架构（Client/Server）

- **两层结构**：客户端 + 数据库服务器
- **特点**：客户端负担重，适合局域网环境
- **优点**：响应速度快、安全性高
- **缺点**：客户端维护困难、扩展性差

### 2.7.2 B/S架构（Browser/Server）

- **三层结构**：浏览器 + Web服务器 + 数据库服务器
- **特点**：客户端为浏览器，跨平台
- **优点**：维护方便、部署简单
- **缺点**：响应速度相对较慢、安全性依赖网络

### 2.7.3 三层架构（Three-Tier）

```
表示层（Presentation Layer）← 用户界面
    ↓
业务逻辑层（Business Logic Layer）← 业务规则与处理
    ↓
数据访问层（Data Access Layer）← 数据库操作
    ↓
数据存储层（Data Storage Layer）← 数据库
```

### 2.7.4 MVC架构模式

- **Model（模型）**：数据与业务逻辑
- **View（视图）**：用户界面
- **Controller（控制器）**：处理输入，协调Model与View

1. TPS、MIS、DSS、ES、OAS的功能定位与区别
2. DSS的组成：数据管理、模型管理、知识管理、用户接口
3. 专家系统的组成：知识库、推理机、解释器、知识获取器
4. C/S架构与B/S架构的特点对比
5. 三层架构各层职责（表示层、业务逻辑层、数据访问层）
6. MVC模式各组件职责
:::

::: warning 重点难点
- **TPS vs MIS**：TPS处理日常事务，MIS提供管理报表；TPS数据来源单一，MIS整合多源数据
- **MIS vs DSS**：MIS处理结构化问题，DSS处理半结构化/非结构化问题；MIS提供定期报告，DSS提供交互式分析
- **专家系统核心**：知识库是核心，推理机是灵魂
- **三层架构设计要点**：各层职责清晰、低耦合、高内聚
:::

# 第五章：数据库设计

> 分值占比：8%-12% | 重要程度：★★★★

## 考情分析

数据库设计是系统架构设计师的核心技能之一。本章涵盖数据库基本概念、关系数据库、数据库设计方法、NoSQL数据库及数据库访问接口等内容。

---

## 5.1 数据库基本概念

### 5.1.1 数据库系统组成

- **数据库（DB）**：长期存储在计算机中的、有组织的、可共享的数据集合
- **数据库管理系统（DBMS）**：管理数据库的软件系统
- **数据库管理员（DBA）**：负责数据库规划、设计、维护
- **数据库应用系统**：基于数据库的应用程序

### 5.1.2 数据模型

- **概念模型**：E-R模型，描述实体与关系
- **逻辑模型**：关系模型、层次模型、网状模型
- **物理模型**：数据在存储介质上的组织方式

### 5.1.3 数据库三级模式结构

- **外模式**：用户视图，用户能看到的数据视图
- **概念模式**：全局逻辑结构，数据库中全部数据的逻辑结构
- **内模式**：物理存储结构，数据在磁盘上的存储方式

### 5.1.4 数据独立性

- **物理独立性**：内模式改变不影响概念模式
- **逻辑独立性**：概念模式改变不影响外模式

## 5.2 关系数据库

### 5.2.1 关系模型基本概念

- **关系（表）**：二维表，由行（元组）和列（属性）组成
- **主键（Primary Key）**：唯一标识一个元组的属性或属性组
- **外键（Foreign Key）**：引用另一个关系主键的属性
- **域**：属性的取值范围

### 5.2.2 SQL语言

**数据定义语言（DDL）：**
- CREATE：创建表、视图、索引
- ALTER：修改表结构
- DROP：删除表、视图

**数据操作语言（DML）：**
- SELECT：查询数据
- INSERT：插入数据
- UPDATE：更新数据
- DELETE：删除数据

**数据控制语言（DCL）：**
- GRANT：授予权限
- REVOKE：撤销权限

### 5.2.3 事务与ACID特性

- **原子性（Atomicity）**：事务是不可分割的最小单元，要么全部成功，要么全部失败
- **一致性（Consistency）**：事务执行前后数据库保持一致性状态
- **隔离性（Isolation）**：并发事务之间相互隔离
- **持久性（Durability）**：事务提交后，修改永久保存

### 5.2.4 数据库完整性

- **实体完整性**：主键不能为空且唯一
- **参照完整性**：外键必须引用存在的记录或为空
- **用户定义完整性**：用户自定义的业务规则

### 5.2.5 数据库并发控制

- **锁机制**：共享锁（S锁）、排他锁（X锁）
- **封锁协议**：一级、二级、三级封锁协议
- **隔离级别**：读未提交、读已提交、可重复读、可串行化

## 5.3 数据库设计（ER模型、规范化）

### 5.3.1 数据库设计步骤

1. 需求分析
2. 概念结构设计（E-R模型）
3. 逻辑结构设计（关系模型）
4. 物理结构设计
5. 数据库实施
6. 数据库运行与维护

### 5.3.2 E-R模型

- **实体（Entity）**：现实世界中可区分的对象
- **属性（Attribute）**：实体的特征
- **联系（Relationship）**：实体之间的关系（1:1、1:N、M:N）

### 5.3.3 关系模式规范化

**规范化目的：** 消除数据冗余，避免插入、删除异常

**各范式：**

| 范式 | 要求 |
|------|------|
| 1NF | 属性不可再分（原子性） |
| 2NF | 满足1NF，非主属性完全函数依赖于主键 |
| 3NF | 满足2NF，非主属性不传递依赖于主键 |
| BCNF | 满足3NF，消除主属性对主键的部分和传递依赖 |
| 4NF | 消除多值依赖 |

## 5.4 NoSQL数据库

### 5.4.1 NoSQL数据库特点

- **非关系型**：不使用传统关系模型
- **分布式**：支持水平扩展
- **灵活模式**：无需预定义表结构
- **高性能**：针对特定场景优化

### 5.4.2 NoSQL数据库分类

- **键值存储（Key-Value）**：Redis、Memcached
- **文档数据库**：MongoDB、CouchDB
- **列族数据库**：HBase、Cassandra
- **图数据库**：Neo4j、JanusGraph

### 5.4.3 NoSQL适用场景

- **键值存储**：缓存、会话管理
- **文档数据库**：内容管理、用户配置
- **列族数据库**：大数据分析、日志数据
- **图数据库**：社交网络、知识图谱

### 5.4.4 NewSQL数据库

NewSQL结合了关系数据库的ACID特性和NoSQL的分布式扩展能力，代表产品：Spanner、TiDB、CockroachDB。

## 5.5 数据库访问接口

### 5.5.1 ODBC（开放数据库连接）

- 统一的数据库访问接口标准
- 应用程序通过ODBC驱动访问不同数据库
- 跨平台、跨数据库

### 5.5.2 JDBC（Java数据库连接）

- Java语言的数据库访问接口
- 提供统一的SQL访问方法
- 支持连接池、事务管理

### 5.5.3 数据库连接池

- **原理**：预先创建并维护一定数量的数据库连接
- **优点**：减少连接创建开销，提高响应速度
- **常见实现**：DBCP、C3P0、Druid、HikariCP

### 5.5.4 ORM框架

- **概念**：对象关系映射（Object-Relational Mapping）
- **作用**：将数据库表映射为对象，简化数据库操作
- **代表框架**：Hibernate、MyBatis、Entity Framework

:::

::: warning 重点难点
- **关系模型核心**：关系就是表，元组就是行，属性就是列
- **规范化目标**：消除数据冗余，避免插入异常和删除异常
- **ACID特性**：数据库事务必须同时满足四个特性
- **锁机制**：S锁允许多个事务读，X锁独占，防止脏读、不可重复读、幻读
- **NoSQL vs 关系数据库**：关系数据库强调一致性与事务，NoSQL强调扩展性与灵活性
:::

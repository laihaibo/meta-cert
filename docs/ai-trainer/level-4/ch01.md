# 第一章：职业道德与法律法规

> 分值占比：15% | 重要程度：★★★★

## 考情分析

本章涵盖人工智能训练师职业伦理规范、数据相关法律法规及行业合规要求。考试侧重对法规条款的记忆与场景判断，职业道德部分常以案例选择题形式出现。掌握本章内容是从事数据标注工作的法律底线。

---

## 1.1 人工智能训练师职业道德

### 1.1.1 职业操守

人工智能训练师应遵守以下基本职业操守：

| 类别 | 具体要求 |
|------|----------|
| 诚信正直 | 如实标注，不篡改数据，不虚报工作量 |
| 客观中立 | 标注不受个人偏见影响，保持一致性 |
| 保密义务 | 对接触的训练数据严格保密，不对外泄露 |
| 持续学习 | 跟进技术发展，提升标注专业能力 |
| 团队协作 | 与算法工程师、项目经理保持有效沟通 |

### 1.1.2 标注工作伦理

**数据偏见防范：**
- 训练数据中的偏见会导致模型输出偏见
- 标注人员应识别并报告明显偏见数据
- 定期轮换标注人员可降低个体偏见影响

**利益冲突：**
- 不得利用训练数据谋取个人利益
- 不得泄露企业或客户的敏感数据

::: tip 高频考点
标注人员发现训练数据含有歧视性内容时，应当：**向项目负责人报告，记录问题，不擅自处理**。这是职业伦理的核心要求。
:::

::: warning 常见陷阱
"标注越快越好" ≠ "标注质量不重要"。四级要求理解准确率与召回率的平衡，质量控制是职责所在。
:::

## 1.2 数据安全与个人信息保护法律法规

### 1.2.1 《中华人民共和国数据安全法》

**核心原则：**
- 数据是国家基础性战略资源
- 数据处理活动应防范风险、保障安全
- 重要数据需进行目录管理
- 数据出口需遵守跨境数据流动规定

**对训练师的要求：**
- 不得非法获取、出售或提供数据
- 发现数据安全事件应立即上报
- 参与数据处理项目前应签署保密协议

### 1.2.2 《中华人民共和国个人信息保护法》（PIPL）

**核心概念：**

| 概念 | 含义 |
|------|------|
| 个人信息 | 以电子或其他方式记录的与已识别或可识别的自然人有关的各种信息 |
| 敏感个人信息 | 生物识别、宗教信仰、特定身份、医疗健康、金融账户、行踪轨迹等 |
| 个人信息处理 | 收集、存储、使用、加工、传输、提供、公开等 |

**处理原则（最小必要）：**
- 目的明确：为特定目的收集，不得超范围使用
- 最小必要：只收集实现目的所必需的最少数据
- 知情同意：个人应被告知并同意数据处理目的
- 安全保护：采取必要措施防止数据泄露

::: tip 高频考点
**"最小必要原则"** 是 PIPL 在数据标注场景中最核心的合规要求。标注前必须确认数据采集目的，不得超出原始授权范围使用数据。
:::

### 1.2.3 《网络安全法》

- 关键信息基础设施运营者需履行网络安全保护义务
- 任何个人和组织不得从事非法侵入他人网络、干扰他人网络正常功能等危害网络安全的活动
- 网络运营者应对收集的用户信息严格保密

## 1.3 知识产权与合规要求

### 1.3.1 训练数据知识产权

**数据来源合规性：**

| 数据来源 | 合规要求 |
|----------|----------|
| 公开数据集 | 需确认开放许可协议（如 CC-BY、MIT 等） |
| 爬虫采集 | 遵守 `robots.txt`，不得突破反爬措施 |
| 用户授权数据 | 需取得用户明示同意，明确使用范围 |
| 采购数据 | 需审查数据供应商的授权链条 |

### 1.3.2 标注成果归属

- 标注成果属于工作产出，通常归委托方所有
- 标注人员不得擅自复制、传播标注数据
- 离职时需交还所有标注数据和工具账号

### 1.3.3 行业合规要点

::: tip 高频考点
**爬虫采集数据的合规边界：**
- 不得违反目标网站的服务协议（Terms of Service）
- 不得干扰目标网站正常运行（发送大量请求导致服务器压力）
- 采集公开数据后二次开发使用需注意原网站许可条款
- 涉及个人信息的爬取需额外满足 PIPL 要求
:::

## 1.4 数据标注中的伦理与合规实践

### 1.4.1 敏感数据识别与处理

**常见敏感数据类型：**

| 类型 | 示例 | 处理要求 |
|------|------|----------|
| 个人身份信息 | 姓名、身份证号、手机号 | 脱敏或排除 |
| 生物特征 | 人脸、指纹、声纹 | 严格脱敏，需专项授权 |
| 金融信息 | 银行卡号、交易记录 | 脱敏，加强访问控制 |
| 医疗健康 | 病历、检查报告 | 严格授权，最小化处理 |

### 1.4.2 标注质量与责任

- 标注人员对标注质量负直接责任
- 发现标注错误应及时更正并上报
- 多人标注需通过交叉验证保证一致性
- 标注争议由项目负责人仲裁

### 1.4.3 数据生命周期管理

```
数据采集 → 数据存储 → 数据处理（标注） → 数据使用 → 数据销毁
    ↑                                                    |
    └──────────────── 安全删除/脱敏 ←───────────────────────┘
```

::: tip 高频考点
**敏感个人信息标注场景：** 当标注任务涉及人脸、语音等生物识别数据时，标注人员应首先确认：项目是否有专项授权文件、数据是否已脱敏、是否在授权范围内操作。三项缺一不可。
:::

## 1.5 常见法律风险与防范

| 风险类型 | 具体情形 | 防范措施 |
|----------|----------|----------|
| 数据泄露 | 标注数据被未授权人员获取 | 权限最小化、数据加密、离职即销权 |
| 知识产权侵权 | 使用未授权数据集 | 审查数据来源授权文件 |
| 个人隐私侵权 | 标注过程中接触到敏感个人信息 | 数据脱敏、签署保密协议 |
| 标注成果侵权 | 标注成果被擅自传播 | 签署知识产权归属协议 |

## 1.6 职业道德案例分析

::: warning 案例分析
**场景：** 某标注人员在标注社交媒体数据时，发现部分帖子包含用户真实姓名、手机号等敏感信息，但项目组未告知需脱敏处理。

**正确做法：**
1. 立即暂停标注该部分数据
2. 向项目负责人报告发现的问题
3. 建议对敏感信息进行脱敏或排除处理
4. 等待项目组明确指示后再继续工作

**错误做法：**
1. 继续标注并记录敏感信息
2. 私下删除敏感数据而未报告
3. 将敏感信息截图分享给同事
:::

## 考点速记

::: tip 高频考点
1. PIPL 核心原则：目的明确、最小必要、知情同意、安全保护
2. 敏感个人信息包括：生物识别、医疗健康、金融账户、行踪轨迹等
3. 数据爬虫合规：遵守 robots.txt、不突破反爬、不干扰网站运行
4. 标注人员职业操守：诚信标注、保守秘密、持续学习
5. 发现数据安全事件应立即上报，不得擅自处理
6. 标注成果归属委托方，离职需交还所有数据
7. 数据生命周期：采集→存储→标注→使用→销毁（全流程安全管理）
:::

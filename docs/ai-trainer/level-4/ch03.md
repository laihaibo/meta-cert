# 第三章：数据标注方法

> 分值占比：35% | 重要程度：★★★★★

## 考情分析

本章是四级考试的核心章节，涵盖四大主流标注类型：分类标注、目标检测、语义分割、文本标注。考试既考察各标注类型的定义、流程、质量要求等理论知识点，也可能以场景题形式考察标注工具的选择与标注规范的制定。实操题占比较高，需熟练掌握至少一种标注工具（Label Studio、CVAT 等）。

---

## 3.1 分类标注

### 3.1.1 分类标注概述

分类标注是最基础的标注类型，为每个数据样本分配一个或多个预定义类别标签。

**核心概念：**

| 概念 | 说明 |
|------|------|
| 二分类 | 样本属于两个类别之一（如 猫/非猫） |
| 多分类 | 样本属于多个互斥类别之一（如 猫/狗/鸟） |
| 多标签分类 | 一个样本可同时属于多个类别（如 图片中同时有"猫"和"室内"） |
| 标签体系 | 类别之间的层级关系（如 动物 → 猫 → 英短） |

### 3.1.2 分类标注流程

```
制定标注规范 → 标注员培训 → 试标注 → 批量标注 → 交叉验证 → 质量评估
```

### 3.1.3 分类标注规范要素

**规范应包含：**
- 类别定义：每个类别的明确描述
- 边界案例处理：模糊样本的归类规则
- 标注工具使用说明
- 争议样本的仲裁流程

### 3.1.4 分类标注质量评估

**核心指标：**

| 指标 | 公式 | 说明 |
|------|------|------|
| 准确率（Precision） | TP / (TP + FP) | 预测为正例中实际为正例的比例 |
| 召回率（Recall） | TP / (TP + FN) | 实际为正例中被正确预测的比例 |
| F1 值 | 2 × P × R / (P + R) | 准确率与召回率的调和平均 |
| 混淆矩阵 | — | 展示各类别的预测结果分布 |
| Cohen's Kappa | — | 考虑随机因素的一致性系数（多标注员时使用） |

**标注一致性评估：**
- 多人标注同一数据集，计算标注间一致性
- 常用指标：Cohen's Kappa（两人）、Fleiss' Kappa（多人）
- Kappa > 0.8 表示高度一致，0.6-0.8 表示中等一致

::: tip 高频考点
**准确率 vs 召回率的权衡：**
- 高准确率要求：宁缺毋滥，减少误报（如垃圾邮件检测）
- 高召回率要求：宁可错杀，不漏过正例（如疾病筛查）
- F1 值综合考虑两者，用于需要平衡的场景
:::

## 3.2 目标检测标注

### 3.2.1 目标检测概述

目标检测标注需要在图像中标注出物体的位置（边界框）和类别。

**标注格式：**

| 格式 | 说明 |
|------|------|
| Bounding Box（边界框） | 矩形框标注物体位置和大小 |
| 类别标签 | 每个边界框对应的物体类别 |
| 置信度（推理时） | 模型对预测结果的置信程度 |

### 3.2.2 边界框标注规范

**标注原则：**
- 框选对象时应紧密贴合物体边缘
- 重叠物体应分别标注（不合并）
- 被遮挡物体：如果主体可见应标注，完全不可见则跳过
- 小物体：应尽量标注，但需设置最小像素阈值

**标注框属性：**

| 属性 | 说明 |
|------|------|
| x_min, y_min | 左上角坐标 |
| x_max, y_max | 右下角坐标 |
| width, height | 宽度和高度 |
| label | 类别标签 |
| difficult | 是否为难例（被遮挡、模糊等） |

### 3.2.3 目标检测评估指标

**IoU（交并比）：**

$$\text{IoU} = \frac{\text{预测框} \cap \text{真实框}}{\text{预测框} \cup \text{真实框}}$$

| IoU 阈值 | 含义 |
|----------|------|
| IoU ≥ 0.5 | 一般认为检测正确 |
| IoU ≥ 0.75 | 检测质量较好 |
| IoU ≥ 0.95 | 检测质量优秀 |

**mAP（均值平均精度）：**

| 概念 | 说明 |
|------|------|
| AP | 单个类别的平均精度（PR曲线下面积） |
| mAP | 所有类别 AP 的平均值 |
| mAP@0.5 | IoU 阈值为 0.5 时的 mAP |
| mAP@0.5:0.95 | IoU 阈值从 0.5 到 0.95 步长 0.05 的 mAP 平均值 |

::: tip 高频考点
**IoU 计算：**
- IoU = 交集面积 / 并集面积
- 当 IoU ≥ 阈值（通常 0.5）时，认为预测框匹配了真实框
- 在标注验收中，标注框与标准框的 IoU 越高，标注质量越好
:::

### 3.2.4 目标检测标注工具

| 工具 | 特点 | 适用场景 |
|------|------|----------|
| Label Studio | 开源，支持多模态 | 通用标注平台 |
| CVAT | 计算机视觉专用 | 图像/视频标注 |
| LabelMe | 轻量级 | 小规模图像标注 |
| VGG Image Annotator | 在线，支持多边形 | 精确标注需求 |

## 3.3 语义分割标注

### 3.3.1 语义分割概述

语义分割需要对图像中的每个像素分配一个类别标签，标注粒度最细。

**三种分割层次：**

| 类型 | 说明 | 标注粒度 |
|------|------|----------|
| 语义分割 | 同类物体不区分实例 | 像素级，同类共享标签 |
| 实例分割 | 区分同一类别的不同实例 | 像素级，每个实例独立 |
| 全景分割 | 结合语义和实例分割 | 像素级，区分所有物体 |

### 3.3.2 标注方法

**多边形标注：**
- 沿物体边缘绘制多边形顶点
- 适用于有明确边界的物体
- 标注精度取决于顶点数量

**画笔标注：**
- 使用画笔工具涂抹区域
- 适用于大面积、不规则区域
- 可使用 eraser 工具修正

**标注规范：**
- 边界像素应尽量贴近物体真实边缘
- 遮挡部分按可见部分标注
- 小物体（<10像素）可简化标注或标注为背景

### 3.3.3 语义分割评估指标

**像素准确率（Pixel Accuracy）：**

$$\text{PA} = \frac{\text{正确分类的像素数}}{\text{总像素数}}$$

**IoU（交并比，像素级）：**

$$\text{IoU} = \frac{\text{预测类和真实类的交集像素数}}{\text{预测类和真实类的并集像素数}}$$

**mIoU（均值 IoU）：**

$$\text{mIoU} = \frac{1}{C} \sum_{i=1}^{C} \text{IoU}_i$$

其中 C 为类别总数。

::: tip 高频考点
**语义分割 vs 目标检测：**
- 语义分割是像素级标注，目标检测是矩形框级标注
- 语义分割标注成本远高于目标检测
- mIoU 是语义分割最重要的评估指标
- 边界像素的标注精度直接影响分割质量
:::

## 3.4 文本标注

### 3.4.1 文本标注类型

| 标注类型 | 说明 | 应用场景 |
|----------|------|----------|
| 文本分类 | 为整段文本分配类别 | 情感分析、主题分类、垃圾邮件检测 |
| 命名实体识别（NER） | 识别并标注文本中的实体 | 人名、地名、组织机构名 |
| 序列标注 | 对文本中每个token标注标签 | 词性标注、分词 |
| 关系抽取 | 标注实体之间的关系 | 知识图谱构建 |
| 情感标注 | 标注文本的情感极性 | 情感分析、舆情监控 |

### 3.4.2 命名实体识别（NER）标注

**常用标注方案（BIO 方案）：**

| 标签 | 说明 | 示例 |
|------|------|------|
| B-PER | 人名开始 | 张 → B-PER |
| I-PER | 人名中间/结尾 | 三 → I-PER |
| B-LOC | 地名开始 | 北 → B-LOC |
| I-LOC | 地名中间/结尾 | 京 → I-LOC |
| B-ORG | 机构名开始 | 腾 → B-ORG |
| I-ORG | 机构名中间/结尾 | 讯 → I-ORG |
| O | 不属于任何实体 | 的 → O |

**示例标注：**

```
原文：张三在北京大学学习
标注：张/B-PER 三/I-PER 在/O 北/B-LOC 京/I-LOC 大/B-ORG 学/I-ORG 学/O 习/O
```

### 3.4.3 文本标注规范

**标注原则：**
- 忠实原文：按文本实际内容标注，不添加主观理解
- 最长匹配：优先标注最长实体（如"北京大学"而非"北京"）
- 嵌套处理：明确是否允许嵌套实体标注
- 歧义处理：多义词按上下文确定含义后标注

### 3.4.4 文本标注评估指标

| 指标 | 说明 |
|------|------|
| 实体准确率 | 标注正确的实体数 / 预测实体总数 |
| 实体召回率 | 标注正确的实体数 / 真实实体总数 |
| 实体 F1 | 准确率与召回率的调和平均 |
| 标签准确率 | 按 token 级别的标注准确率 |

::: tip 高频考点
**BIO 标注方案：**
- B（Begin）：实体起始位置
- I（Inside）：实体中间或结束位置
- O（Outside）：非实体
- 同一实体的连续 token 必须用 B 开头，后续用 I 标记
- B-* 和 I-* 的 * 必须对应（如 B-PER 后只能跟 I-PER）
:::

## 3.5 标注工具使用

### 3.5.1 Label Studio 基础

**项目创建流程：**

```
1. 安装 Label Studio（Docker 或 pip）
2. 创建项目 → 设置项目名称
3. 配置标注任务（导入数据、选择标注类型）
4. 邀请标注人员 → 分配任务
5. 开始标注 → 导出结果
```

**支持的标注类型：**

| 标注类型 | Label Studio 配置 |
|----------|-------------------|
| 文本分类 | `<TextClassification>` |
| NER | `<Labels>` with `<Text>` |
| 图像分类 | `<ImageClassification>` |
| 目标检测 | `<RectangleLabels>` |
| 语义分割 | `<PolygonLabels>` |

### 3.5.2 标注项目质量管理

**三审流程：**

| 审核层级 | 审核内容 | 通过标准 |
|----------|----------|----------|
| 一审（自检） | 标注员自查 | 无遗漏、无错标 |
| 二审（互检） | 另一标注员交叉检查 | 一致率 ≥ 90% |
| 三审（终审） | 质检员抽检 | 抽检准确率 ≥ 95% |

**质量回溯：**
- 一审不通过：打回标注员重新标注
- 二审不通过：上报项目负责人仲裁
- 连续不达标标注员：需重新培训

::: warning 常见陷阱
**标注边界模糊：** 对于模糊样本（如图片中猫和狗难以区分），不应猜测，应按标注规范中的"边界案例处理规则"执行，或标记为"需仲裁"。
:::

## 考点速记

::: tip 高频考点
1. 分类标注：多分类 vs 多标签（互斥 vs 非互斥）
2. 目标检测：IoU = 交集 / 并集，mAP 为核心评估指标
3. 语义分割：mIoU 为核心评估指标，标注粒度最细
4. 文本标注：NER 使用 BIO 方案（B-PER、I-PER、O）
5. 标注质量：三审流程（自检→互检→终审），终检准确率 ≥ 95%
6. 标注工具：Label Studio 支持多模态标注，CVAT 专注计算机视觉
7. 标注一致性：Cohen's Kappa > 0.8 为高度一致
8. 混淆矩阵：TP（真阳性）、FP（假阳性）、FN（假阴性）、TN（真阴性）
:::

::: warning 计算公式
- **IoU：** 交集面积 / 并集面积
- **Precision：** TP / (TP + FP)
- **Recall：** TP / (TP + FN)
- **F1：** 2 × P × R / (P + R)
- **mIoU：** 各类别 IoU 的平均值
- **Cohen's Kappa：** (Po - Pe) / (1 - Pe)，Po 为观测一致性，Pe 为期望随机一致性
:::

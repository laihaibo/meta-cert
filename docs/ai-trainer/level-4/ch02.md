## 2.1 数据采集方法

### 2.1.1 数据来源分类

| 数据来源 | 说明 | 典型应用场景 |
|----------|------|--------------|
| 公开数据集 | 政府、学术机构、企业公开的数据集 | 通用模型预训练、基准测试 |
| 网络爬虫 | 从网页中自动提取数据 | 舆情分析、商品信息采集 |
| 传感器采集 | 摄像头、麦克风、IoT 设备等 | 自动驾驶、智能语音 |
| API 接口 | 调用第三方服务接口获取数据 | 社交媒体数据、地图数据 |
| 人工录入 | 纸质/口头数据的人工数字化 | 古籍数字化、表单数字化 |

### 2.1.2 网络爬虫基础

**基本流程：**

```
发送 HTTP 请求 → 解析 HTML/JSON → 提取目标数据 → 存储
```

**核心要素：**

| 要素 | 说明 |
|------|------|
| User-Agent | 标识请求来源，部分网站会校验 |
| 请求间隔 | 避免高频请求被封 IP，建议设置延时 |
| robots.txt | 网站爬虫规则文件，应遵守 |
| 代理 IP | 大规模采集时使用，防止被封 |

::: tip 高频考点
**爬虫采集的合规要求（PIPL相关）：**
- 遵守目标网站的 robots.txt 和服务条款
- 不得使用技术手段突破网站反爬措施
- 采集个人数据需满足"最小必要"和"知情同意"原则
- 不得干扰目标网站的正常运行
:::

### 2.1.3 API 数据获取

**常见 API 类型：**

| API 类型 | 协议 | 特点 |
|----------|------|------|
| REST API | HTTP/HTTPS | 基于资源，使用 HTTP 方法（GET/POST/PUT/DELETE） |
| GraphQL API | POST | 按需查询，减少数据冗余 |
| WebSocket API | WebSocket | 实时双向通信 |

**API 请求基本要素：**
- 请求 URL（包含路径和查询参数）
- 请求头（Headers）：API Key、Content-Type、Authorization
- 请求体（Body）：POST 请求携带的数据
- 响应格式：通常为 JSON

### 2.1.4 传感器数据采集

**常见传感器类型：**

| 传感器 | 采集数据类型 | 典型应用 |
|--------|-------------|----------|
| 摄像头 | 图像、视频 | 目标检测、人脸识别 |
| 麦克风 | 音频 | 语音识别、声纹识别 |
| 激光雷达（LiDAR） | 3D 点云 | 自动驾驶、三维建模 |

## 2.2 数据清洗流程

### 2.2.1 数据清洗概述

数据清洗是指发现并纠正数据中错误、不一致和不完整的过程，是数据预处理的核心环节。

```
原始数据 → 数据加载 → 缺失值处理 → 重复值处理 → 异常值处理 → 格式标准化 → 清洗后数据
```

### 2.2.2 缺失值处理

**缺失值类型：**

| 类型 | 说明 | 示例 |
|------|------|------|
| 完全随机缺失（MCAR） | 缺失与任何值无关 | 系统随机丢包 |
| 随机缺失（MAR） | 缺失与已观测值有关 | 女性更可能不填写"吸烟史" |
| 非随机缺失（MNAR） | 缺失与缺失值本身有关 | 低收入者不愿填写收入 |

**处理方法：**

| 方法 | 适用场景 | 说明 |
|------|----------|------|
| 删除法 | 缺失率 < 5%，样本量充足 | 直接删除含缺失值的记录 |
| 均值/中位数填充 | 数值型，数据分布较对称 | 用该字段均值/中位数替代 |
| 众数填充 | 分类型 | 用出现频率最高的值替代 |
| 前向/后向填充 | 时序数据 | 用前一个/后一个有效值替代 |
| 模型预测填充 | 高价值数据 | 用其他特征训练模型预测缺失值 |

### 2.2.3 重复值处理

**重复值检测：**
- 全字段完全重复：所有字段值完全相同
- 主键重复：唯一标识字段重复（如用户 ID）
- 部分字段重复：关键字段重复（如手机号）

**去重策略：**

| 策略 | 说明 | 适用场景 |
|------|------|----------|
| 保留第一条 | 出现重复时保留首次出现的记录 | 默认策略 |
| 保留最后一条 | 出现重复时保留最后出现的记录 | 以最新数据为准 |
| 保留非空值 | 合并重复记录，保留有值的字段 | 数据补全场景 |
| 全部删除 | 所有重复记录均删除 | 数据质量要求极高 |

### 2.2.4 异常值处理

**异常值检测方法：**

| 方法 | 原理 | 适用场景 |
|------|------|----------|
| Z-Score | 超过均值 ±3 倍标准差 | 近似正态分布 |
| IQR 法 | 超出 Q1-1.5IQR 或 Q3+1.5IQR | 数据分布未知 |
| 箱线图法 | 可视化识别 | 快速筛查 |
| 孤立森林 | 基于树模型检测 | 高维数据 |

**异常值处理策略：**

| 策略 | 说明 |
|------|------|
| 删除 | 确认是错误数据后删除 |
| 修正 | 可推断出正确值时进行修正 |
| 截断/盖帽 | 将极端值限制在合理范围内 |
| 单独标记 | 保留并标记为异常，供后续分析 |

::: tip 高频考点
**IQR 法异常值判断：**
- 下界 = Q1 - 1.5 × IQR（IQR = Q3 - Q1）
- 上界 = Q3 + 1.5 × IQR
- 超出此范围的值视为异常值
:::

### 2.2.5 格式标准化

**常见标准化需求：**

| 字段类型 | 标准化内容 |
|----------|------------|
| 日期时间 | 统一格式（如 YYYY-MM-DD），处理时区 |
| 数值 | 统一单位（如 kg/lb、℃/℉），处理异常精度 |
| 文本 | 去除首尾空格、统一大小写、处理全角/半角 |
| 分类值 | 统一枚举值（如 "男"/"男性"/"M" → "男"） |
| 编码 | 统一字符编码（UTF-8） |

### 2.2.6 数据清洗 Python 实操（Pandas）

**常用操作：**

```python
import pandas as pd

# 1. 加载数据
df = pd.read_csv('data.csv', encoding='utf-8')

# 2. 缺失值处理
df['age'].fillna(df['age'].median(), inplace=True)  # 中位数填充
df.dropna(subset=['label'], inplace=True)            # 删除 label 缺失

# 3. 重复值处理
df.drop_duplicates(subset=['id'], keep='first', inplace=True)

# 4. 异常值处理（IQR法）
Q1 = df['score'].quantile(0.25)
Q3 = df['score'].quantile(0.75)
IQR = Q3 - Q1
df = df[(df['score'] >= Q1 - 1.5*IQR) & (df['score'] <= Q3 + 1.5*IQR)]

# 5. 格式标准化
df['name'] = df['name'].str.strip()              # 去空格
df['category'] = df['category'].str.lower()      # 统一小写
```

::: tip 高频考点
**数据清洗检查清单：**
1. 缺失值：统计各字段缺失率，制定填充/删除策略
2. 重复值：根据主键或关键字段去重
3. 异常值：用统计方法检测并合理处理
4. 格式：统一日期、数值、文本格式
5. 一致性：检查跨字段逻辑关系（如结束时间不能早于开始时间）
:::

## 2.3 数据质量控制

### 2.3.1 数据质量维度

| 维度 | 定义 | 评估方法 |
|------|------|----------|
| 完整性 | 数据是否缺失 | 缺失率统计 |
| 准确性 | 数据是否正确反映真实情况 | 抽样人工校验、与权威数据比对 |
| 一致性 | 数据是否在多个来源间一致 | 跨源比对、冲突检测 |
| 时效性 | 数据是否在有效时间范围内 | 时间戳检查、过期数据比例 |
| 唯一性 | 数据是否存在重复 | 重复率统计 |
| 有效性 | 数据是否符合预定义格式/范围 | 格式校验、范围校验 |

### 2.3.2 数据质量评估指标

**关键指标：**

| 指标 | 公式 | 说明 |
|------|------|------|
| 数据完整率 | 完整记录数 / 总记录数 × 100% | 衡量数据缺失程度 |
| 数据准确率 | 正确记录数 / 总记录数 × 100% | 抽样人工校验 |
| 数据一致率 | 一致记录数 / 总记录数 × 100% | 跨系统/跨字段比对 |
| 重复数据率 | 重复记录数 / 总记录数 × 100% | 衡量数据冗余 |
| 异常数据率 | 异常记录数 / 总记录数 × 100% | 统计方法检测 |

### 2.3.3 质量控制的实施流程

```
制定质量标准 → 数据质量检测 → 问题数据标记 → 数据清洗修复 → 质量复检 → 质量报告
```

**质量控制角色：**

| 角色 | 职责 |
|------|------|
| 数据采集员 | 按规范采集原始数据 |
| 数据清洗员 | 执行数据清洗流程 |
| 质检员 | 抽样检查数据质量，标记问题 |
| 项目经理 | 审批质量标准，处理争议问题 |

### 2.3.4 抽样检验方法

**简单随机抽样：**
- 每个样本被抽中的概率相等
- 适用于数据分布均匀的场景

**分层抽样：**
- 按类别/特征分层后在每层内随机抽样
- 适用于类别不平衡的数据集

**抽样比例确定：**

| 总体规模 | 建议抽样比例 | 最小样本量 |
|----------|--------------|------------|
| < 1,000 | 10%-20% | 100 |
| 1,000-10,000 | 5%-10% | 200 |
| 10,000-100,000 | 2%-5% | 300 |
| > 100,000 | 1%-2% | 500 |

### 2.3.5 数据质量报告

**报告要素：**
- 数据总量与各字段统计概览
- 各质量维度指标（完整率、准确率、一致率等）
- 发现的问题数据清单及处理建议
- 清洗前后质量对比

::: tip 高频考点
**数据质量控制核心关注：**
1. 清洗前必须记录数据质量问题基线（各字段缺失率、重复率）
2. 清洗后需复检，确认质量达标
3. 质量报告应包含清洗前后对比数据
4. 抽样检验需保证样本具有代表性
:::

## 2.4 数据存储与管理

### 2.4.1 数据存储格式

| 格式 |  ｜文件类型 | 特点 | 适用场景 |
|------|----------|------|----------|
| CSV | 文本 | 通用，易读写，不支持复杂结构 | 表格数据，小规模数据交换 |
| JSON | 文本 | 支持嵌套结构，可读性好 | 结构化/半结构化数据 |
| Parquet | 二进制 | 列式存储，压缩率高 | 大数据分析，列查询 |
| TFRecord | 二进制 | TensorFlow 专用，高效读取 | 深度学习训练数据 |
| XML | 文本 | 标记严格，体积大 | 遗留系统，文档类数据 |

### 2.4.2 数据版本管理

- 原始数据需存档，不可直接修改
- 清洗后数据应记录清洗操作日志
- 使用版本号或 Git 管理数据变更历史
- 标注数据需区分训练集/验证集/测试集

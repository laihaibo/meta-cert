## 3.1 训练流程

### 3.1.1 标准训练流程

**完整训练流程：**

```
准备数据 → 构建模型 → 配置损失函数与优化器 → 训练循环 → 验证评估 → 调优迭代 → 测试上线
```

**各阶段详细说明：**

| 阶段 | 主要工作 | 关键要点 |
|------|----------|----------|
| 数据准备 | 加载、预处理、划分数据集 | 数据质量、分布一致性 |
| 模型构建 | 选择架构、初始化参数 | 架构适配任务、初始化策略 |
| 配置优化器 | 选择损失函数、优化器 | 任务匹配、超参数设置 |
| 训练循环 | 前向传播、计算损失、反向传播、更新权重 | 批次大小、训练轮数 |
| 验证评估 | 在验证集上评估指标 | 监控过拟合、早停判断 |
| 调优迭代 | 调整超参数、修改架构 | 网格搜索、随机搜索、贝叶斯优化 |
| 测试上线 | 在测试集上最终评估 | 性能达标、部署准备 |

### 3.1.2 训练 epochs 与 batch size

**Epoch（训练轮数）：**
- 定义：完整遍历一次训练数据集
- 设置原则：
  - 太少：欠拟合，模型未充分学习
  - 太多：过拟合，模型记住训练数据
- 常用范围：10-100轮，结合早停策略

**Batch Size（批次大小）：**
- 定义：每次参数更新所用的样本数
- 常见设置：32、64、128、256（2的幂次）
- 影响：
  - 大批次：训练稳定，梯度准确，内存需求大
  - 小批次：噪声梯度，可能跳出局部最优，内存需求小

### 3.1.3 学习率设置

**学习率（Learning Rate）的作用：**
- 控制每次参数更新的步长
- 过大：可能导致不收敛或震荡
- 过小：收敛速度慢，可能陷入局部最优

**学习率调度策略：**
- 固定学习率：简单任务
- 学习率衰减：训练后期减小学习率
  - Step Decay：每隔N轮衰减为原来的γ倍
  - Exponential Decay：指数衰减
  - Cosine Annealing：余弦退火
- 自适应学习率：Adam、RMSProp等优化器自动调整

::: tip 高频考点
1. 标准训练流程的七个阶段
2. Epoch定义：完整遍历一次训练数据集
3. Batch Size常用2的幂次：32、64、128、256
4. 学习率过大导致震荡，过小导致收敛慢
5. 早停策略可防止过拟合
:::

## 3.2 超参数调整

### 3.2.1 超参数分类

**模型超参数（训练前设置）：**
- 网络层数、每层神经元数量
- 学习率、批次大小、训练轮数
- 正则化系数、Dropout比例

**算法超参数（与算法选择相关）：**
- 优化器类型、损失函数类型
- 卷积核大小、池化方式

### 3.2.2 超参数搜索方法

**网格搜索（Grid Search）：**
- 遍历所有超参数组合
- 优点：简单直观，不遗漏最佳组合
- 缺点：维度灾难，计算量随参数数量指数增长

**随机搜索（Random Search）：**
- 在参数空间中随机采样
- 优点：计算效率高，高维空间表现更好
- 缺点：可能遗漏最优组合

**贝叶斯优化（Bayesian Optimization）：**
- 基于概率模型引导搜索方向
- 优点：样本效率高，自动平衡探索与利用
- 缺点：实现复杂，单次评估成本高

**自动化工具：**
- Hyperopt
- Optuna
- Ray Tune

### 3.2.3 常用超参数调优策略

| 策略 | 适用场景 | 方法 |
|------|----------|------|
| 粗调 | 初步确定范围 | 大范围、大步长搜索 |
| 精调 | 缩小最优范围 | 小范围、小步长搜索 |
| 学习率范围测试 | 确定合适学习率 | 从极小到极大逐渐增大 |
| 早停 | 防止过拟合 | 验证集性能不再提升时停止 |

::: warning 调优原则
- 先调整影响大的超参数（学习率、网络结构）
- 再调整影响小的超参数（Dropout、权重衰减）
- 每次只调整一个超参数，观察影响
- 保留调优记录，便于复现与对比
- 使用验证集评估，测试集仅用于最终评估
:::

## 3.3 损失函数

### 3.3.1 损失函数基本概念

**定义：**
- 衡量模型预测值与真实值之间差距的函数
- 训练目标：最小化损失函数

**常用损失函数分类：**
- 回归任务损失
- 分类任务损失
- 自定义损失

### 3.3.2 回归任务损失函数

| 损失函数 | 公式 | 特点 |
|----------|------|------|
| 均方误差（MSE） | Σ(y-ŷ)²/n | 最常用，对大误差敏感 |
| 平均绝对误差（MAE） | Σ|y-ŷ|/n | 对异常值鲁棒 |
| Huber损失 | MSE（小误差），MAE（大误差） | 结合两者优点 |

### 3.3.3 分类任务损失函数

**二分类损失：**

| 损失函数 | 公式 | 特点 |
|----------|------|------|
| 二元交叉熵（Binary Cross Entropy） | -Σ[y·log(ŷ)+(1-y)·log(1-ŷ)] | 最常用 |
| Hinge Loss | max(0, 1-y·ŷ) | SVM使用，支持向量机 |

**多分类损失：**

| 损失函数 | 公式 | 特点 |
|----------|------|------|
| 交叉熵损失（Cross Entropy） | -Σy_i·log(ŷ_i) | 最常用，信息论角度 |
| KL散度 | Σp(x)·log(p(x)/q(x)) | 衡量分布差异 |

### 3.3.4 损失函数选择原则

**选择依据：**
- 任务类型：回归、二分类、多分类
- 数据特点：是否含异常值、类别是否平衡
- 模型输出：最后一层激活函数

**常见搭配：**
- 回归任务 + 线性输出 → MSE损失
- 二分类任务 + Sigmoid输出 → 二元交叉熵
- 多分类任务 + Softmax输出 → 交叉熵损失

::: tip 高频考点
1. 损失函数衡量预测值与真实值的差距
2. 回归任务常用MSE，对异常值敏感
3. 分类任务常用交叉熵损失
4. 二元交叉熵用于二分类，交叉熵用于多分类
5. MAE对异常值比MSE鲁棒
6. 损失函数需与模型输出激活函数匹配
:::

## 3.4 优化器

### 3.4.1 优化器基本概念

**定义：**
- 用于更新模型参数以最小化损失函数的算法
- 基于梯度下降原理

**梯度下降原理：**
- 计算损失相对于参数的梯度
- 沿梯度反方向更新参数
- 参数更新：θ = θ - η × ∇J(θ)

### 3.4.2 常见优化器对比

| 优化器 | 核心思想 | 优点 | 缺点 | 适用场景 |
|--------|----------|------|------|----------|
| SGD | 随机梯度下降 | 简单，内存占用小 | 震荡大，收敛慢 | 小数据集、简单任务 |
| Momentum | 引入动量项 | 加速收敛，减少震荡 | 需调动量系数 | 凸优化、局部极值 |
| AdaGrad | 自适应学习率，累加梯度平方 | 稀疏数据表现好 | 学习率过早衰减 | NLP、稀疏特征 |
| RMSProp | 指数加权平均梯度平方 | 解决AdaGrad衰减问题 | 需调衰减系数 | RNN、非平稳目标 |
| Adam | Momentum + RMSProp | 综合优势，收敛快 | 内存占用稍大 | 通用场景，默认选择 |
| AdamW | Adam + 权重衰减修正 | 泛化性更好 | 实现稍复杂 | 深度学习通用 |

### 3.4.3 优化器选择指南

**选择建议：**
- **默认首选**：Adam，综合性能好
- **追求泛化**：AdamW（带权重衰减修正）
- **内存受限**：SGD + Momentum
- **稀疏数据**：AdaGrad
- **RNN/时序任务**：RMSProp或Adam
- **精细调优**：SGD + 动量 + 学习率衰减

**优化器超参数：**
- 学习率（lr）：最重要的超参数
- Beta1、Beta2（Adam）：控制动量和RMSProp的衰减
- Epsilon：防止除零的小常数
- 权重衰减：L2正则化系数

### 3.4.4 优化器工作原理详解

**SGD（随机梯度下降）：**
```
v = η × ∇J(θ)
θ = θ - v
```

**Momentum：**
```
v = β × v_prev + η × ∇J(θ)
θ = θ - v
```
- β：动量系数，通常0.9
- 累积历史梯度方向，加速收敛

**Adam（自适应矩估计）：**
```
m = β1 × m_prev + (1-β1) × ∇J(θ)  # 一阶矩估计
v = β2 × v_prev + (1-β2) × ∇J(θ)²  # 二阶矩估计
m_hat = m / (1-β1^t)               # 偏差修正
v_hat = v / (1-β2^t)               # 偏差修正
θ = θ - η × m_hat / (√v_hat + ε)   # 参数更新
```

::: warning 常见误区
- Adam并非在所有任务上都优于SGD，SGD配合良好学习率调度仍可能获得更好泛化
- 学习率是最重要的超参数，优先调优
- AdamW比Adam泛化性更好，建议优先使用
- 权重衰减≠L2正则化，AdamW做了修正
:::

## 3.5 正则化与防止过拟合

### 3.5.1 L1与L2正则化

**L1正则化（Lasso）：**
- 损失函数加上权重的L1范数
- 公式：Loss = 原损失 + λ × Σ|w|
- 特点：产生稀疏权重，可用于特征选择

**L2正则化（Ridge/Weight Decay）：**
- 损失函数加上权重的L2范数
- 公式：Loss = 原损失 + λ × Σw²
- 特点：权重趋向于较小值，防止过拟合

### 3.5.2 Dropout

**原理：**
- 训练时随机"丢弃"部分神经元
- 防止神经元之间的共适应
- 相当于训练多个子网络的集成

**使用方法：**
- 通常在全连接层后使用
- Dropout比例：0.2-0.5
- 测试时不使用Dropout，输出乘以保留概率

### 3.5.3 早停（Early Stopping）

**原理：**
- 监控验证集性能
- 当验证集性能连续N轮不提升时停止训练
- 防止过拟合

**参数设置：**
- Patience：连续不提升的轮数（通常5-20轮）
- 监控指标：验证集损失或准确率
- 恢复最佳权重：保存验证集性能最好的模型

### 3.5.4 数据增强

**图像数据增强：**
- 几何变换：旋转、翻转、裁剪、缩放
- 色彩变换：亮度、对比度、饱和度调整
- 噪声注入：高斯噪声、椒盐噪声

**文本数据增强：**
- 同义词替换
- 随机插入、交换、删除
- 回译（中英互译）

## 3.6 模型评估与选择

### 3.6.1 模型评估流程

```
训练集训练 → 验证集调参 → 测试集评估 → 选择最优模型
```

**数据集职责：**
- 训练集：学习模型参数
- 验证集：选择模型、调整超参数
- 测试集：最终评估模型性能（仅使用一次）

### 3.6.2 模型选择策略

| 策略 | 方法 | 适用场景 |
|------|------|----------|
| 验证集性能 | 选择验证集性能最好的模型 | 标准做法 |
| 交叉验证 | 多折交叉验证的平均性能 | 数据量较小 |
| 集成方法 | 多个模型投票或平均 | 追求极致性能 |

### 3.6.3 泛化能力评估

**泛化误差分解：**
- 泛化误差 = 偏差 + 方差 + 噪声
- 偏差：模型预测值与真实值的平均差距（欠拟合）
- 方差：模型预测值的波动程度（过拟合）
- 噪声：数据本身的不可约误差

**学习曲线分析：**
- 训练曲线上升，验证曲线平稳：欠拟合
- 训练曲线远高于验证曲线：过拟合
- 两者接近且性能高：拟合良好

# 第三章：模型优化与评估

> 分值占比：约35% | 重要程度：★★★★★

## 考情分析

模型优化与评估是二级/技师考试的核心章节。考生需要深入理解模型评估指标体系，掌握超参数优化和模型部署的基本方法。考试既考查概念理解，也涉及计算方法（如混淆矩阵、各类指标公式），实操题常要求根据给定的评估结果判断模型优劣并给出优化建议。

---

## 3.1 模型评估指标

### 3.1.1 分类模型评估指标

**混淆矩阵（Confusion Matrix）：**

|  | 预测为正例 | 预测为负例 |
|--|----------|----------|
| 实际为正例 | TP（真阳性） | FN（假阴性） |
| 实际为负例 | FP（假阳性） | TN（真阴性） |

**核心指标：**

| 指标 | 公式 | 含义 |
|------|------|------|
| 准确率（Accuracy） | (TP + TN) / (TP + TN + FP + FN) | 整体预测正确的比例 |
| 精确率（Precision） | TP / (TP + FP) | 预测为正例中实际为正例的比例 |
| 召回率（Recall） | TP / (TP + FN) | 实际为正例中被正确预测的比例 |
| F1分数 | 2 × Precision × Recall / (Precision + Recall) | 精确率和召回率的调和平均 |
| 特异度（Specificity） | TN / (TN + FP) | 实际为负例中被正确预测的比例 |

**ROC曲线与AUC：**
- **ROC曲线**：以假正率（FPR）为横轴，真正率（TPR）为纵轴绘制
- **AUC（曲线下面积）**：ROC曲线下的面积，取值范围[0, 1]
- AUC越接近1，模型区分能力越强；AUC=0.5相当于随机猜测

### 3.1.2 回归模型评估指标

| 指标 | 公式 | 含义 |
|------|------|------|
| MAE（平均绝对误差） | Σ\|y_i - ŷ_i\| / n | 预测值与真实值绝对误差的平均 |
| MSE（均方误差） | Σ(y_i - ŷ_i)² / n | 预测值与真实值误差平方的平均 |
| RMSE（均方根误差） | √MSE | MSE的平方根，与原始数据同量级 |
| MAPE（平均绝对百分比误差） | Σ\|y_i - ŷ_i\| / y_i / n × 100% | 百分比形式的误差 |
| R²（决定系数） | 1 - SS_res / SS_tot | 模型解释的数据变异比例 |

### 3.1.3 模型评估注意事项

**类别不平衡问题：**
- 准确率在不平衡数据上具有误导性
- 应同时关注精确率、召回率、F1分数
- 可使用混淆矩阵进行详细分析

**过拟合与欠拟合：**
- **过拟合**：模型在训练集表现好，在测试集表现差（高方差）
- **欠拟合**：模型在训练集和测试集表现都差（高偏差）
- **理想状态**：偏差和方差都较低

## 3.2 超参数优化

### 3.2.1 超参数与模型参数

**模型参数：**
- 通过训练数据学习得到（如神经网络权重、线性回归系数）
- 不需要人工设置

**超参数：**
- 在训练前人工设置的参数
- 常见超参数：
  - 学习率（Learning Rate）
  - 批大小（Batch Size）
  - 训练轮数（Epochs）
  - 正则化系数（如L1/L2的λ）
  - 树模型的最大深度、最小叶子样本数
  - KNN的K值、SVM的C和γ

### 3.2.2 超参数优化方法

**网格搜索（Grid Search）：**
- 对超参数的所有组合进行穷举搜索
- 优点：简单直观，不遗漏最优组合
- 缺点：计算成本高，维度灾难

**随机搜索（Random Search）：**
- 在超参数空间中随机采样
- 优点：比网格搜索更高效，尤其在高维空间
- 缺点：可能错过最优组合

**贝叶斯优化（Bayesian Optimization）：**
- 基于已有评估结果建立代理模型（如高斯过程）
- 根据采集函数选择下一个评估点
- 优点：样本效率高，适合计算成本高的场景
- 缺点：实现复杂

**常见优化策略：**
- 学习率衰减：初始大学习率快速下降，后期小学习率精细调整
- 早停（Early Stopping）：验证集性能不再提升时停止训练
- 交叉验证：使用K折交叉验证评估超参数组合效果

### 3.2.3 超参数调优实践

**调优顺序建议：**
1. 先调整影响最大的超参数（如学习率、模型复杂度）
2. 使用粗粒度搜索找到大致范围
3. 在较优范围内进行细粒度搜索
4. 使用交叉验证确保结果稳定

## 3.3 模型部署

### 3.3.1 部署模式

| 部署模式 | 特点 | 适用场景 |
|----------|------|----------|
| 在线部署（Online） | 实时响应，低延迟 | 实时推荐、实时风控 |
| 批量部署（Batch） | 定时批量处理 | 离线评分、批量预测 |
| 边缘部署（Edge） | 在终端设备运行 | 移动端、IoT设备 |
| 流式部署（Streaming） | 持续处理数据流 | 实时数据管道、异常检测 |

### 3.3.2 部署考虑因素

**性能指标：**
- **延迟（Latency）**：请求响应时间
- **吞吐量（Throughput）**：每秒处理请求数
- **并发能力**：同时处理多个请求的能力

**可扩展性：**
- 水平扩展：增加机器数量
- 垂直扩展：提升单机配置

**监控与维护：**
- 模型性能监控：定期评估模型效果
- 数据漂移检测：监控输入数据分布变化
- 概念漂移检测：监控目标变量分布变化
- 模型版本管理：追踪模型迭代历史

### 3.3.3 模型上线流程

1. **模型验证**：在独立测试集上验证模型性能
2. **性能测试**：测试延迟、吞吐量等性能指标
3. **A/B测试**：与现有模型对比效果
4. **灰度发布**：先小流量验证，再逐步扩大
5. **监控告警**：建立监控体系，异常时告警
6. **回滚机制**：出现问题时可快速回退

### 3.3.4 模型迭代

**触发迭代的条件：**
- 模型性能下降到阈值以下
- 数据分布发生显著变化（数据漂移）
- 业务场景发生变化
- 新技术/新算法出现

**迭代周期：**
- 监控阶段：持续监控模型表现
- 评估阶段：定期评估是否需要迭代
- 重新训练：使用新数据重新训练
- 验证测试：在新数据上验证效果
- 上线发布：通过验证后上线

8. 数据漂移和概念漂移的概念
:::

::: warning 计算公式
- Precision = TP / (TP + FP)
- Recall = TP / (TP + FN)
- F1 = 2 × P × R / (P + R)
- FPR = FP / (FP + TN)
- TPR = TP / (TP + FN)（即Recall）
- R² = 1 - SS_res / SS_tot，其中 SS_res = Σ(y_i - ŷ_i)²，SS_tot = Σ(y_i - ȳ)²
:::

::: tip 选择题常考
在不平衡数据集中，仅使用准确率评估模型是不合适的，应同时关注精确率、召回率和F1分数。
:::

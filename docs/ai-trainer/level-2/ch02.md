# 第二章：业务分析与设计

> 分值占比：约35% | 重要程度：★★★★★

## 考情分析

业务分析与设计是二级/技师考试的核心模块，要求考生能够将业务需求转化为技术方案，设计合理的业务流程和数据指标体系。考试重点考查需求分析能力、流程设计能力和指标体系构建能力。案例分析题通常以实际业务场景为背景，要求考生提出完整的解决方案。

---

## 2.1 业务流程设计

### 2.1.1 业务流程分析基础

**核心概念：**
- **流程（Process）**：为达到特定目标而进行的一系列有逻辑关联的活动
- **活动（Activity）**：流程中的最小工作单元
- **节点（Node）**：流程中的关键决策点或处理点
- **泳道（Swimlane）**：表示不同角色或部门在流程中的职责范围

### 2.1.2 业务流程建模方法

| 建模方法 | 适用场景 | 优点 |
|----------|----------|------|
| 流程图（Flowchart） | 简单线性流程 | 直观易懂，便于沟通 |
| BPMN（业务流程模型符号） | 复杂业务流程 | 标准化程度高，支持自动化 |
| 事件驱动流程链（EPC） | 企业级流程建模 | 表达能力强，适合复杂决策 |
| 数据流图（DFD） | 数据密集型流程 | 清晰展示数据流向和转换 |

### 2.1.3 常见业务流程模式

**顺序模式：**
```
开始 → 步骤1 → 步骤2 → 步骤3 → 结束
```

**分支模式：**
```
开始 → 条件判断 → 分支A/分支B → 结束
```

**循环模式：**
```
开始 → 循环体（满足条件时执行） → 结束
```

**并行模式：**
```
开始 → [分支A] / [分支B]（并行执行） → 汇聚 → 结束
```

### 2.1.4 业务流程优化

**优化原则：**
- **消除浪费**：识别并消除不增值的活动
- **简化流程**：减少不必要的审批环节
- **自动化**：将重复性工作自动化
- **并行化**：将串行活动改为并行执行
- **标准化**：统一流程模板，减少变异

## 2.2 数据分析方法

### 2.2.1 数据分析基础

**数据分析层次：**
- **描述性分析**：发生了什么？（数据报表、数据看板）
- **诊断性分析**：为什么发生？（根因分析、相关性分析）
- **预测性分析**：将发生什么？（机器学习、时间序列预测）
- **规范性分析**：应该怎么做？（优化算法、决策支持）

### 2.2.2 常用分析方法

| 分析方法 | 适用场景 | 输出结果 |
|----------|----------|----------|
| 对比分析 | 不同群体/时间维度的比较 | 差异报告、趋势图表 |
| 漏斗分析 | 转化路径优化（如用户转化漏斗） | 各环节转化率、流失率 |
| 留存分析 | 用户留存情况 | 留存曲线、留存率 |
|  cohort 分析 | 用户分群行为研究 | 不同群组的长期表现 |
| 关联分析 | 发现变量间的关系（如购物篮分析） | 关联规则、置信度、提升度 |
| 聚类分析 | 用户分群、异常检测 | 用户画像、异常样本 |
| 回归分析 | 预测连续值 | 预测模型、影响因素权重 |

### 2.2.3 数据采集与预处理

**数据采集方式：**
- **埋点采集**：在应用/网站中埋设代码采集用户行为
- **日志采集**：采集服务器日志、应用日志
- **API采集**：通过接口获取第三方数据
- **爬虫采集**：抓取公开网页数据

**数据清洗步骤：**
1. **缺失值处理**：删除、填充（均值/中位数/众数）、预测填充
2. **异常值处理**：识别（3σ原则、箱线图）、处理（删除/修正/分箱）
3. **重复值处理**：识别并删除重复记录
4. **格式标准化**：统一日期、时间、编码格式
5. **数据一致性检查**：验证跨表数据的一致性

### 2.2.4 特征工程

**特征类型：**
- **数值型特征**：连续值（如年龄、收入）
- **类别型特征**：离散值（如性别、职业）
- **时序特征**：时间相关的特征（如用户活跃时间）
- **文本特征**：自然语言描述（如评论、描述）
- **图像特征**：视觉相关特征

**特征处理方法：**
| 特征类型 | 处理方法 | 说明 |
|----------|----------|------|
| 数值型 | 标准化/归一化 | 消除量纲差异 |
| 数值型 | 分箱 | 将连续值转换为区间类别 |
| 类别型 | One-Hot编码 | 适用于类别较少的情况 |
| 类别型 | Label编码 | 适用于有序类别 |
| 类别型 | 目标编码 | 用目标变量的统计值编码 |
| 文本型 | TF-IDF/Word2Vec/BERT | 将文本转换为数值向量 |

## 2.3 指标体系构建

### 2.3.1 指标体系设计原则

**SMART原则：**
- **Specific（具体）**：指标定义清晰明确
- **Measurable（可衡量）**：可通过数据计算
- **Achievable（可实现）**：数据可获取，计算可行
- **Relevant（相关性）**：与业务目标紧密相关
- **Time-bound（时效性）**：有明确的时间周期

### 2.3.2 常见指标体系

**电商核心指标体系：**

| 维度 | 一级指标 | 二级指标 |
|------|----------|----------|
| 流量 | 访问量（UV/PV） | 新访客占比、跳出率、平均停留时长 |
| 转化 | 转化率 | 加购率、下单率、支付率 |
| 营收 | GMV | 客单价、ARPU、复购率 |
| 用户 | 活跃用户数 | DAU/MAU、留存率、流失率 |

**内容平台指标体系：**

| 维度 | 一级指标 | 二级指标 |
|------|----------|----------|
| 内容消费 | 消费次数/时长 | 人均阅读数、阅读完成率 |
| 内容生产 | 发布量 | 创作者数、人均发布量 |
| 互动 | 互动率 | 点赞率、评论率、分享率 |
| 用户 | 活跃度 | 留存率、使用频率 |

**客户服务指标体系：**

| 维度 | 一级指标 | 二级指标 |
|------|----------|----------|
| 服务效率 | 响应时间 | 首次响应时间、平均解决时间 |
| 服务质量 | 满意度 | CSAT、NPS、投诉率 |
| 服务量 | 服务量 | 工单量、接通率 |
| 运营 | 运营效率 | 人均处理量、成本 |

### 2.3.3 指标拆解方法

**OSM模型：**
- **O（Objective）**：业务目标是什么？
- **S（Strategy）**：用什么策略实现目标？
- **M（Measurement）**：如何衡量策略效果？

**AARRR模型（海盗模型）：**
- **Acquisition（获取）**：用户从哪里来
- **Activation（激活）**：用户首次良好体验
- **Retention（留存）**：用户持续使用
- **Revenue（营收）**：用户付费
- **Referral（推荐）**：用户推荐他人

### 2.3.4 指标体系搭建步骤

1. **明确业务目标**：确定核心目标和关键结果
2. **识别关键指标**：通过OSM模型推导核心指标
3. **构建指标树**：从一级指标逐层拆解到可采集的二级、三级指标
4. **确定数据来源**：明确每个指标的数据采集方式和计算口径
5. **建立监控体系**：设计数据看板，实现指标实时监控
6. **持续迭代优化**：根据业务变化调整指标体系

7. OSM模型和AARRR模型的应用场景
:::

::: warning 易混淆点
- 对比分析和关联分析的区别：前者比较不同群体，后者发现变量间关系
- 漏斗分析和 cohort 分析的区别：漏斗看转化路径，cohort看用户群长期行为
- One-Hot编码适用于无序类别，Label编码适用于有序类别
- OSM模型从目标出发推导指标，AARRR模型从用户生命周期出发
:::

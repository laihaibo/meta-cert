# 高频考点速查

> 本章内容涵盖二级/技师考试的高频考点，适合考前快速回顾和查漏补缺。

---

## 一、职业道德与法律法规

### 法律要点速记

| 法律法规 | 核心要求 |
|----------|----------|
| 《数据安全法》 | 数据分类分级保护，重要数据本地化存储 |
| 《个人信息保护法》 | 合法正当必要诚信原则，敏感信息需单独同意 |
| 《算法推荐管理规定》 | 算法备案，不得歧视，一键关闭个性化推荐 |
| 《生成式AI管理暂行办法》 | 训练数据合法，服务提供者承担主体责任 |
| 《AI伦理规范》 | 公平公正、透明可释、隐私保护、可控可信 |

### 易考点

- 敏感个人信息的处理必须取得**单独同意**
- 数据脱敏方法：掩码、替换、泛化、加噪、抑制
- 算法备案条件：具有**舆论属性或社会动员能力**

## 二、业务分析与设计

### 数据分析方法

| 分析方法 | 核心用途 |
|----------|----------|
| 描述性分析 | 发生了什么（报表、看板） |
| 诊断性分析 | 为什么发生（根因分析） |
| 预测性分析 | 将发生什么（机器学习） |
| 规范性分析 | 应该怎么做（优化决策） |

### 特征处理方法速记

| 特征类型 | 处理方法 | 备注 |
|----------|----------|------|
| 数值型 | 标准化/归一化 | 消除量纲差异 |
| 类别型（无序） | One-Hot编码 | 类别数不宜过多 |
| 类别型（有序） | Label编码 | 保留顺序关系 |
| 文本型 | TF-IDF/Word2Vec/BERT | 转换为数值向量 |

### 指标体系设计

- **SMART原则**：具体、可衡量、可实现、相关、有时效
- **OSM模型**：目标 → 策略 → 衡量
- **AARRR模型**：获取、激活、留存、营收、推荐

## 三、模型优化与评估

### 分类指标公式速查

| 指标 | 公式 |
|------|------|
| 准确率 | (TP + TN) / (TP + TN + FP + FN) |
| 精确率 | TP / (TP + FP) |
| 召回率 | TP / (TP + FN) |
| F1分数 | 2 × P × R / (P + R) |
| FPR | FP / (FP + TN) |
| TPR | TP / (TP + FN) = Recall |

### 回归指标速记

| 指标 | 特点 |
|------|------|
| MAE | 绝对值平均，对异常值敏感 |
| MSE | 平方平均，放大了大误差 |
| RMSE | MSE开方，与原始数据同量级 |
| R² | 越接近1越好，表示模型解释力 |

### 超参数优化方法

| 方法 | 特点 |
|------|------|
| 网格搜索 | 穷举所有组合，准确但慢 |
| 随机搜索 | 随机采样，效率更高 |
| 贝叶斯优化 | 基于历史结果优化，样本效率高 |

### 模型部署模式

| 模式 | 适用场景 |
|------|----------|
| 在线部署 | 实时推荐、实时风控 |
| 批量部署 | 离线评分、批量预测 |
| 边缘部署 | 移动端、IoT设备 |
| 流式部署 | 实时数据管道 |

### 关键概念

- **过拟合**：训练集好，测试集差 → 解决方案：正则化、早停、增加数据
- **欠拟合**：训练集和测试集都差 → 解决方案：增加模型复杂度、更多特征
- **数据漂移**：输入数据分布变化
- **概念漂移**：目标变量分布变化

## 四、综合考点

### 常考题型

1. **概念辨析**：区分易混淆概念（如精确率 vs 召回率，数据漂移 vs 概念漂移）
2. **指标计算**：根据混淆矩阵计算准确率、精确率、召回率
3. **场景选择**：给定业务场景选择合适的分析方法或部署模式
4. **法律法规判断**：根据场景判断行为是否合规
5. **流程设计**：根据业务需求设计合理的数据处理或模型训练流程

### 考前检查清单

- [ ] 熟练掌握混淆矩阵及各类评估指标的计算
- [ ] 理解过拟合与欠拟合的区别及解决方法
- [ ] 熟记常见法律法规的核心要点
- [ ] 掌握数据清洗的基本步骤
- [ ] 理解指标体系设计方法（SMART、OSM、AARRR）
- [ ] 了解超参数优化的主要方法
- [ ] 掌握模型部署模式的选择依据

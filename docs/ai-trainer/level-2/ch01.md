# 第一章：职业道德与法律法规

> 分值占比：约10% | 重要程度：★★★

## 考情分析

职业道德与法律法规是各级别考试的公共模块。二级/技师级别要求考生在熟悉基础法律规范的基础上，进一步理解人工智能领域的数据安全、个人信息保护、算法伦理等专项法律法规，并能在实际业务中正确应用。

## 1.1 职业道德规范

### 1.1.1 人工智能训练师职业道德基本要求

**核心原则：**
- **诚信守法**：遵守国家法律法规和行业规范
- **客观公正**：在数据标注、模型评估中保持中立和客观
- **保护隐私**：严格保护用户数据和隐私信息
- **持续学习**：跟踪技术发展，不断提升专业能力
- **责任意识**：对工作成果负责，主动发现并报告问题

### 1.1.2 数据标注中的伦理规范

**关键要求：**
- 数据标注应忠实于原始数据，不随意篡改
- 标注结果应保持一致性和可复现性
- 发现有害内容应立即上报
- 遵守数据保密协议，不对外泄露训练数据

## 1.2 相关法律法规

### 1.2.1 《中华人民共和国数据安全法》

**核心要点：**
- 国家建立数据分类分级保护制度
- 重要数据需进行本地化存储
- 开展数据处理活动应建立数据安全管理制度
- 发生数据安全事件时应立即采取补救措施并报告

### 1.2.2 《中华人民共和国个人信息保护法》

**核心要点：**
- 处理个人信息应遵循合法、正当、必要、诚信原则
- 收集个人信息应明确告知处理目的、方式、范围
- 敏感个人信息需取得单独同意
- 个人有权查询、更正、删除其个人信息
- 自动化决策应保证决策的透明度和结果的公平公正

### 1.2.3 《互联网信息服务算法推荐管理规定》

**核心要点：**
- 算法推荐服务提供者应建立完备的算法安全机制
- 不得利用算法实施歧视
- 应设置一键关闭、删除账号等便捷功能
- 用户可选择关闭个性化推荐
- 算法备案制度：具有舆论属性或社会动员能力的算法推荐服务需备案

### 1.2.4 《生成式人工智能服务管理暂行办法》

**核心要点：**
- 生成式AI服务应遵守法律法规，尊重社会公德
- 训练数据应合法，不得侵犯知识产权
- 服务提供者应建立投诉举报机制
- 生成内容应进行安全评估和过滤
- 服务提供者对生成内容承担主体责任

### 1.2.5 《人工智能伦理规范》

**核心要点：**
- 增进人类福祉：AI发展应以人类利益为根本出发点
- 公平公正：避免算法偏见和歧视
- 透明可释：AI决策过程应可解释、可追溯
- 隐私保护：充分保障个人隐私和数据安全
- 可控可信：AI系统应处于人类可控范围内

## 1.3 数据安全与隐私保护

### 1.3.1 数据分类与分级

**数据分类：**
- 个人信息：可识别特定自然人的数据
- 重要数据：一旦泄露可能危害国家安全和公共利益的数据
- 核心数据：关系国家核心利益的关键数据

**分级保护：**
- 一般数据：常规保护措施
- 重要数据：加强保护措施，本地化存储
- 核心数据：最高级别保护

### 1.3.2 数据脱敏技术

| 脱敏方法 | 适用场景 | 说明 |
|----------|----------|------|
| 掩码（Masking） | 身份证、手机号 | 用星号或特定字符替换部分信息 |
| 替换（Substitution） | 姓名、地址 | 用虚构但格式一致的内容替换 |
| 泛化（Generalization） | 年龄、收入 | 将精确值转换为区间 |
| 加噪（Noise Addition） | 数值型数据 | 添加随机噪声保护原始值 |
| 抑制（Suppression） | 极度敏感字段 | 直接删除或忽略该字段 |

### 1.3.3 跨境数据传输

**合规要求：**
- 向境外提供重要数据需通过安全评估
- 个人信息出境需满足以下条件之一：
  - 取得个人单独同意
  - 通过个人信息保护认证
  - 签订标准合同
  - 通过网信部门的安全评估

---

## 1.4 算法合规与伦理审查

### 1.4.1 算法备案制度

**备案范围：**
- 具有舆论属性或社会动员能力的算法推荐服务
- 深度合成服务
- 其他具有生成式人工智能特征的服务

**备案要求：**
- 算法名称、基本原理
- 算法目的、应用场景
- 数据使用情况
- 安全评估报告

### 1.4.2 算法偏见与公平性

**常见偏见来源：**
- 训练数据中的历史偏见
- 特征选择中的主观偏见
- 模型设计中的隐含偏见

**缓解措施：**
- 数据层面：均衡数据集，去偏处理
- 特征层面：审查特征与敏感属性的关联
- 模型层面：引入公平性约束
- 评估层面：分群组评估模型性能

## 考点速记

::: tip 高频考点
1. 《个人信息保护法》中敏感个人信息的处理要求（单独同意）
2. 数据分类：个人信息、重要数据、核心数据
3. 算法推荐服务需备案的条件（舆论属性/社会动员能力）
4. 数据脱敏的主要方法及其适用场景
5. 生成式AI服务提供者的主体责任
6. 算法公平性问题的来源与缓解措施
:::

::: warning 易混淆点
- "重要数据"不等同于"个人信息"，但个人信息中可能包含重要数据
- 算法备案仅适用于具有特定性质的算法服务，不是所有AI算法都需要备案
- 跨境数据传输的限制主要是针对"重要数据"和达到一定规模的"个人信息"
:::

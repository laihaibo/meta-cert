# 第二章：业务战略与创新

> 分值占比：30%-40% | 重要程度：★★★★★

## 考情分析

本章是**一级考试的核心章节**，分值占比最高。要求考生具备从业务战略高度规划 AI 产品的能力，包括指标体系设计、技术选型评估、NLP 和 CV 领域的业务应用创新。考试题型偏重综合分析题和方案设计题。

## 2.1 业务战略规划

### 2.1.1 AI 产品战略定位

AI 产品战略规划的核心是回答三个问题：
- **为什么做** — 业务痛点和价值主张
- **做什么** — AI 能力边界和功能范围
- **怎么做** — 技术路径和实施路线

**战略分析框架（SWOT 分析）：**

| 维度 | 内容 |
|------|------|
| 优势（Strengths） | 数据积累、算法能力、技术团队 |
| 劣势（Weaknesses） | 数据质量、标注成本、人才缺口 |
| 机会（Opportunities） | 政策支持、市场需求、技术进步 |
| 威胁（Threats） | 竞争加剧、监管趋严、技术迭代 |

### 2.1.2 AI 成熟度评估

组织 AI 成熟度通常分为五个阶段：

| 阶段 | 特征 |
|------|------|
| 初始级 | 无明确 AI 战略，零散试点 |
| 发展级 | 有 AI 规划，但缺乏系统性 |
| 定义级 | 建立 AI 治理框架，有明确路线图 |
| 量化级 | AI 价值可量化度量，持续优化 |
| 优化级 | AI 驱动业务创新，形成核心竞争力 |

::: tip 高频考点
AI 产品从试点到规模化的五个阶段是战略规划的基础框架，需结合案例理解每个阶段的特征和关键动作。
:::

### 2.1.3 商业模式设计

AI 产品的典型商业模式：

| 模式 | 说明 | 示例 |
|------|------|------|
| API 服务 | 按调用量收费 | 语音识别 API、NLP API |
| SaaS 订阅 | 按用户/功能订阅 | 智能客服 SaaS |
| 项目定制 | 针对客户需求定制 | 企业智能分析系统 |
| 数据增值 | 基于数据洞察收费 | 用户画像、精准营销 |
| 平台生态 | 连接供需双方 | AI 模型市场 |

## 2.2 指标体系设计

### 2.2.1 AI 产品指标体系框架

一级/高级技师需要掌握 AI 产品全生命周期的指标体系设计：

```
业务指标层（Why）
    ↓
产品指标层（What）
    ↓
模型指标层（How）
```

**三层指标体系：**

| 层级 | 指标类型 | 示例 |
|------|----------|------|
| 业务指标 | 用户增长、收入、转化率 | DAU、GMV、转化率 |
| 产品指标 | 功能使用率、满意度 | 功能渗透率、NPS |
| 模型指标 | 准确率、召回率、F1 | 准确率、精确率、延迟 |

### 2.2.2 模型评估核心指标

**分类任务指标：**

| 指标 | 公式 | 说明 |
|------|------|------|
| 准确率（Accuracy） | (TP+TN)/(TP+TN+FP+FN) | 预测正确的比例 |
| 精确率（Precision） | TP/(TP+FP) | 预测为正例中实际为正例的比例 |
| 召回率（Recall） | TP/(TP+FN) | 实际为正例中被正确预测的比例 |
| F1 分数 | 2·P·R/(P+R) | 精确率和召回率的调和平均 |
| AUC | ROC 曲线下面积 | 综合衡量分类器性能 |

其中：TP（真正例）、TN（真反例）、FP（假正例）、FN（假反例）

**回归任务指标：**

| 指标 | 公式 | 说明 |
|------|------|------|
| MAE | 平均绝对误差 | 预测值与真实值差的绝对值均值 |
| MSE | 均方误差 | 差的平方均值 |
| RMSE | 均方根误差 | MSE 的平方根 |
| R² | 决定系数 | 模型解释方差的比例 |

**多标签/多分类指标：**
- 宏平均（Macro）：各类指标算术平均
- 微平均（Micro）：先汇总 TP/FP/FN，再计算指标
- 加权平均（Weighted）：按各类样本数加权

### 2.2.3 指标体系设计方法

**北极星指标（North Star Metric）：**
- 唯一核心指标，反映产品为用户创造的核心价值
- 示例：电商平台的"月度购买用户数"

**OMTM（One Metric That Matters）：**
- 当前阶段唯一最重要的指标
- 每个周期（周/月）确定一个 OMTM，团队聚焦

**指标体系建设步骤：**

1. 明确业务目标 → 2. 定义用户旅程 → 3. 确定关键行为 → 4. 设计衡量指标 → 5. 建立指标体系

::: tip 高频考点
1. 准确率 vs 精确率 vs 召回率的区别和适用场景
2. F1 分数在精确率和召回率不平衡时的作用
3. 北极星指标和 OMTM 的概念
4. 三层指标体系（业务→产品→模型）的层级关系
5. 混淆矩阵（TP/TN/FP/FN）的计算和含义
:::

::: warning 易错点
在样本不平衡场景下，准确率（Accuracy）可能产生误导，此时应关注精确率、召回率和 F1 分数。
:::

## 2.3 自然语言处理与计算机视觉业务应用

### 2.3.1 自然语言处理（NLP）

NLP 是人工智能训练师最重要的应用领域之一。

**NLP 核心技术栈：**

| 技术方向 | 核心技术 | 典型应用 |
|----------|----------|----------|
| 文本分类 | BERT、RoBERTa | 新闻分类、垃圾邮件识别 |
| 命名实体识别 | BiLSTM-CRF、BERT | 人名/地名/机构名提取 |
| 信息抽取 | 关系抽取、事件抽取 | 知识图谱构建 |
| 文本生成 | GPT、T5 | 对话系统、内容生成 |
| 语义理解 | 语义相似度、阅读理解 | 智能问答、搜索引擎 |
| 机器翻译 | Transformer | 多语言翻译 |
| 情感分析 | 情感分类模型 | 舆情监控、评论分析 |

**大语言模型（LLM）在业务中的应用：**

| 应用场景 | 实现方式 | 效果评估 |
|----------|----------|----------|
| 智能客服 | 微调/提示工程 | 问题解决率、转人工率 |
| 文档处理 | 信息提取+生成 | 准确率、效率提升 |
| 代码辅助 | 代码补全/审查 | 代码采纳率、bug率 |
| 知识问答 | RAG/微调 | 回答准确率、引用正确率 |
| 内容创作 | 提示工程/微调 | 人工编辑率、内容多样性 |

**NLP 模型训练关键要点：**
1. **数据质量** > 数据量：高质量标注数据比大量低质量数据更重要
2. **领域适配**：通用大模型需在特定领域数据上微调
3. **提示工程**：善用 few-shot、CoT（思维链）等提示策略
4. **评估体系**：建立业务相关的人工评估+自动指标双轨评估

### 2.3.2 计算机视觉（CV）

**CV 核心技术栈：**

| 技术方向 | 核心技术 | 典型应用 |
|----------|----------|----------|
| 图像分类 | ResNet、ViT | 商品分类、缺陷检测 |
| 目标检测 | YOLO、Faster R-CNN | 人脸检测、车辆检测 |
| 图像分割 | U-Net、Mask R-CNN | 医学影像、自动驾驶 |
| 人脸识别 | FaceNet、ArcFace | 门禁考勤、金融核身 |
| OCR | CRNN、DBNet | 文档识别、车牌识别 |
| 图像生成 | GAN、Stable Diffusion | 图像修复、数据增强 |

**多模态 AI 应用：**

| 应用 | 输入 | 输出 | 核心技术 |
|------|------|------|----------|
| 图文理解 | 图片+文本 | 文本描述 | CLIP、BLIP |
| 视频分析 | 视频流 | 行为识别/事件标注 | I3D、TimeSformer |
| 视觉问答 | 图片+问题 | 文本答案 | VQA 模型 |
| 数字人 | 文本/语音 | 视频/图像 | TTS + 人脸生成 |

**CV 数据标注关键要点：**
1. **标注规范**：制定详细的标注指南，确保标注一致性
2. **质量控制**：采用多人标注+交叉验证+抽检机制
3. **难例处理**：建立难例上报和专家评审机制
4. **数据增强**：合理使用翻转、旋转、裁剪、颜色抖动等增强策略
5. **分布管理**：确保训练/验证/测试集分布一致

::: tip 高频考点
1. NLP 核心技术分类和应用场景（文本分类、NER、文本生成等）
2. 大语言模型应用方式：微调 vs 提示工程 vs RAG 的适用场景
3. CV 核心技术分类（分类、检测、分割、OCR）
4. 多模态 AI 的定义和应用场景
5. NLP/CV 模型训练中的数据质量优先原则
:::

::: warning 重要提示
大模型（LLM）和多模态 AI 是近年考试重点，需重点关注提示工程、RAG、微调等工程化实践。
:::

### 2.3.3 AI 业务应用选型决策

当企业需要引入 AI 能力时，需要考虑以下因素：

| 考量维度 | 要点 |
|----------|------|
| 业务需求 | 问题定义清晰，ROI 可预期 |
| 数据基础 | 数据量、数据质量、标注可行性 |
| 技术选型 | 开源模型 vs 商用 API vs 自研 |
| 成本评估 | 数据成本、算力成本、人力成本、维护成本 |
| 合规风险 | 数据安全、算法备案、行业监管 |
| 团队能力 | 算法能力、工程能力、运维能力 |

**技术选型决策树：**

```
是否有高质量标注数据？
├── 否 → 考虑少样本/零样本方案，或先做数据积累
└── 是 → 精度要求是否极高？
    ├── 否 → 考虑预训练模型微调
    └── 是 → 是否需要领域深度定制？
        ├── 否 → 商用 API 或通用模型
        └── 是 → 自主训练或联合研发
```

## 2.4 模型创新方法

### 2.4.1 技术选型与评估

**开源模型评估维度：**

| 维度 | 评估内容 |
|------|----------|
| 性能 | 基准测试（MMLU、C-Eval、COCO 等） |
| 效率 | 推理速度、显存占用、模型大小 |
| 生态 | 社区活跃度、文档完善度、工具链 |
| 商业 | 许可证（Apache/MIT/BSD vs 非商用） |
| 兼容 | 框架兼容性（PyTorch/TensorFlow/ONNX） |

### 2.4.2 A/B 测试与效果验证

AI 功能上线的标准流程：

```
数据准备 → 模型开发 → 离线评估 → A/B 测试 → 全量上线
```

**离线评估：**
- 在独立测试集上评估模型指标
- 与基线模型（Baseline）对比
- 分析 bad case，定位问题

**A/B 测试：**
- 对照组：现有方案
- 实验组：新 AI 方案
- 核心指标：业务指标变化
- 统计显著性：确保结果可靠

**灰度发布策略：**

| 阶段 | 流量比例 | 目标 |
|------|----------|------|
| 小流量 | 1%-5% | 验证系统稳定性 |
| 中流量 | 10%-20% | 观察业务指标 |
| 大流量 | 50% | 全面验证 |
| 全量 | 100% | 正式上线 |

### 2.4.3 持续学习与迭代

AI 模型不是一次性工程，需要持续优化：

```
监控反馈 → Bad Case 分析 → 数据补充 → 模型迭代 → 效果验证
    ↑                                                      |
    └────────────────────────────────────────────────────────┘
```

**持续学习的关键能力：**
1. 自动化监控系统：追踪模型漂移（Drift）
2. 反馈闭环：用户反馈自动回流到训练数据
3. 版本管理：模型版本、数据版本、配置版本一体化管理
4. 快速迭代：CI/CD 流水线支持模型快速迭代和上线

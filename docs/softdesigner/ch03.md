# 第三章：数据结构

> 分值占比：15%-20% | 重要程度：★★★★★

## 考情分析

数据结构是软件设计师考试的核心章节，分值占比最高。考试重点包括线性结构、树与二叉树、图、排序与查找算法的原理、复杂度及应用场景。需要熟练掌握各种数据结构的特性、存储方式、操作实现及算法复杂度分析。

---

## 线性结构

### 数组

- **定义**：一段连续存储的相同类型数据元素的序列
- **存储方式**：顺序存储，物理地址连续
- **操作**：随机访问（O(1)）、插入（O(n)）、删除（O(n)）
- **特点**：读快写慢，空间利用率高

### 链表

- **单链表**：每个节点含数据域和指针域，只指向后继
- **双链表**：每个节点含前驱指针和后继指针
- **循环链表**：尾节点指针指向头节点，构成环
- **操作复杂度**：
  - 随机访问：O(n)
  - 插入/删除（已知位置）：O(1)
  - 插入/删除（未知位置）：O(n)（需先查找）

### 稀疏矩阵

- **定义**：矩阵中多数元素为零，仅存储非零元素
- **三元组表示法**：(行号, 列号, 值)
- **压缩存储**：节省存储空间，便于矩阵运算

---

## 栈与队列

### 栈

- **定义**：限定仅在表尾进行插入和删除的线性表（后进先出，LIFO）
- **存储方式**：顺序栈（数组）、链栈（链表）
- **基本操作**：Push（入栈）、Pop（出栈）、Peek（查看栈顶）
- **应用场景**：表达式求值、括号匹配、函数调用栈、递归转非递归

### 队列

- **定义**：限定在表头删除、表尾插入的线性表（先进先出，FIFO）
- **存储方式**：
  - **顺序队列**：数组实现，存在"假溢出"问题
  - **循环队列**：首尾相连，解决假溢出
  - **链队列**：链表实现
- **基本操作**：Enqueue（入队）、Dequeue（出队）
- **应用场景**：缓冲区、消息队列、广度优先遍历、层次遍历

### 双端队列（Deque）

- **定义**：允许在两端进行插入和删除的线性表
- **输入受限双端队列**：一端只能插入，另一端可插入删除
- **输出受限双端队列**：一端只能删除，另一端可插入删除

---

## 串

### 串的基本操作

- **定义**：由零个或多个字符组成的有限序列
- **存储方式**：
  - **定长顺序存储**：固定长度数组
  - **堆分配存储**：动态分配，灵活可变
  - **块链存储**：链表方式存储
- **基本操作**：赋值、连接、求子串、定位（模式匹配）

### KMP算法

- **目的**：高效的模式串匹配算法
- **核心思想**：利用已匹配部分的信息，避免不必要的回溯
- **Next数组**：记录模式串中每个位置的最长公共前后缀长度
- **时间复杂度**：O(n+m)，n为主串长度，m为模式串长度
- **优势**：相比暴力匹配的 O(n×m) 有显著提升

## 数组与广义表

### 数组

- **多维数组的存储**：按行优先或按列优先存储
- **行优先（C/Java）**：行下标变化慢，列下标变化快
- **列优先（Fortran）**：列下标变化慢，行下标变化快
- **地址计算**：数组元素地址 = 基地址 + 下标偏移量×元素大小

### 广义表

- **定义**：是线性表的推广，表中元素可以是原子也可以是子表
- **表示**：LS = (a₁, a₂, ..., aₙ)，其中每个aᵢ可以是原子或广义表
- **存储方式**：链式存储（头尾链表表示法）
- **深度**：广义表展开后括号嵌套的最大层数
- **长度**：最外层元素个数
- **空表**：广义表中不含任何元素，记作 ()

## 树与二叉树

### 二叉树的基本性质

- **性质1**：第i层最多有 2^(i-1) 个节点（i≥1）
- **性质2**：深度为k的二叉树最多有 2^k - 1 个节点（k≥1）
- **性质3**：叶子节点数 n₀ = n₂ + 1（n₂为度为2的节点数）
- **性质4**：具有n个节点的完全二叉树深度 ⌈log₂(n+1)⌉
- **性质5**：对完全二叉树，若节点从1编号，则：
  - 节点i的左孩子编号为 2i
  - 节点i的右孩子编号为 2i+1
  - 节点i的父节点编号为 ⌊i/2⌋

### 二叉树遍历

- **前序遍历**（根-左-右）： NLR
- **中序遍历**（左-根-右）： LNR
- **后序遍历**（左-右-根）： LRN
- **层序遍历**：逐层从左到右
- **遍历特点**：
  - 已知前序+中序可唯一确定二叉树
  - 已知后序+中序可唯一确定二叉树
  - 仅已知前序+后序不能唯一确定（存在多解）

### 二叉排序树（BST）

- **定义**：左子树所有节点值 < 根节点值 < 右子树所有节点值
- **中序遍历结果**：有序序列（升序）
- **查找效率**：平均 O(log n)，最坏 O(n)（退化为链表）
- **插入/删除**：保持BST性质

### 平衡二叉树（AVL树）

- **定义**：任意节点的左右子树高度差不超过1（平衡因子 ∈ {-1, 0, 1}）
- **平衡因子**：左子树高度 - 右子树高度
- **目的**：避免BST退化为链表，保证操作复杂度为 O(log n)
- **失衡类型**：
  - LL型：左子树的左子树插入 → 右旋
  - RR型：右子树的右子树插入 → 左旋
  - LR型：左子树的右子树插入 → 先左旋再右旋
  - RL型：右子树的左子树插入 → 先右旋再左旋

### 哈夫曼树

- **定义**：带权路径长度（WPL）最小的二叉树
- **构建方法**：贪心算法，每次选取最小权值的两棵树合并
- **带权路径长度 WPL** = Σ(Wi × Li)，Wi为叶子权值，Li为路径长度
- **哈夫曼编码**：
  - 前缀编码：任一字符的编码都不是另一字符编码的前缀
  - 压缩效率高，出现频率高的字符编码短
- **特点**：没有度为1的节点（n个叶子节点，共2n-1个节点）

### 树与森林的转换

- **森林转二叉树**：兄弟相连，父节点保留最左孩子的链接
- **二叉树转森林**：左分支恢复原树，右分支恢复原兄弟

## 图

### 图的存储

- **邻接矩阵**：n×n矩阵，A[i][j]=1表示i到j有边
  - 无向图：对称矩阵，空间复杂度 O(n²)
  - 有向图：不一定对称
- **邻接表**：每个顶点关联一个链表，存储其所有邻接点
  - 空间复杂度 O(n+e)，适合稀疏图
  - 查找边：O(e)（需要遍历链表）

### 图的遍历

- **深度优先遍历（DFS）**：递归或栈实现，一条路走到头再回溯
  - 时间复杂度 O(n+e)
- **广度优先遍历（BFS）**：队列实现，逐层扩展
  - 时间复杂度 O(n+e)

### 最小生成树

- **Prim算法**：加点法，从某个顶点出发，每次选择与已选顶点集连接的最小边
  - 时间复杂度 O(n²)，适合稠密图
- **Kruskal算法**：加边法，按边权从小到大选择，不形成环
  - 时间复杂度 O(e log e)，适合稀疏图

### 最短路径

- **Dijkstra算法**：单源最短路径，贪心策略
  - 不能处理带负权边
  - 时间复杂度 O(n²)
- **Floyd算法**：多源最短路径，动态规划
  - 可处理带负权边（不能有负环）
  - 时间复杂度 O(n³)
- **Bellman-Ford算法**：单源，可处理负权边，检测负环

### 拓扑排序

- **适用条件**：有向无环图（DAG）
- **定义**：将图中所有顶点排成一个线性序列，使得任意有向边(u, v)，u在v之前
- **算法**：
  1. 选择入度为0的顶点输出
  2. 删除该顶点及其出边
  3. 重复直到所有顶点输出或存在环
- **应用**：工程进度安排、任务调度

### 关键路径

- **前提**：AOE网（带权有向无环图），边表示活动，顶点表示事件
- **关键路径**：路径长度（总权值）最长的路径，决定工程最短完成时间
- **关键活动**：位于关键路径上的活动，延迟会影响总工期
- **最早开始时间 Ve(j)**：从源点到j的最长路径长度
- **最迟开始时间 Vl(j)**：不影响总工期的最晚开始时间
- **活动余量**：Slack = Vl - Ve，为0的活动是关键活动

## 排序算法

### 插入排序

- **直接插入排序**：将未排序元素逐个插入到已排序序列的合适位置
  - 时间复杂度：最好 O(n)，最坏 O(n²)，平均 O(n²)
  - 空间复杂度：O(1)
  - 稳定性：稳定
- **折半插入排序**：查找插入位置时用二分法
  - 时间复杂度：查找 O(log n)，移动仍 O(n)，整体 O(n²)
  - 稳定性：稳定

### 选择排序

- **直接选择排序**：每次从未排序部分选最小元素放到已排序部分末尾
  - 时间复杂度：O(n²)
  - 空间复杂度：O(1)
  - 稳定性：不稳定（可能改变相同元素的相对顺序）
- **堆排序**：利用堆（优先队列）进行排序
  - 建堆：O(n)，排序：O(n log n)
  - 空间复杂度：O(1)
  - 稳定性：不稳定

### 交换排序

- **冒泡排序**：相邻元素比较，逆序则交换，每趟确定一个最大值
  - 时间复杂度：最好 O(n)（优化版），最坏 O(n²)
  - 空间复杂度：O(1)
  - 稳定性：稳定
- **快速排序**：选取基准元素，分区排序，递归处理
  - 时间复杂度：平均 O(n log n)，最坏 O(n²)
  - 空间复杂度：O(log n)（递归栈）
  - 稳定性：不稳定

### 归并排序

- **二路归并排序**：将序列分成两半，分别排序后合并
  - 时间复杂度：O(n log n)
  - 空间复杂度：O(n)
  - 稳定性：稳定

### 基数排序

- **思想**：从低位到高位（或高位到低位），按位值分配到桶中
- **时间复杂度**：O(d×(n+r))，d为位数，r为基数
- **空间复杂度**：O(n+r)
- **稳定性**：稳定
- **非比较排序**，不受O(n log n)下限限制

### 排序算法复杂度对比

| 排序算法 | 平均时间 | 最坏时间 | 最好时间 | 空间复杂度 | 稳定性 |
|---------|---------|---------|---------|-----------|--------|
| 冒泡排序 | O(n²) | O(n²) | O(n) | O(1) | 稳定 |
| 直接选择 | O(n²) | O(n²) | O(n²) | O(1) | 不稳定 |
| 直接插入 | O(n²) | O(n²) | O(n) | O(1) | 稳定 |
| 快速排序 | O(n log n) | O(n²) | O(n log n) | O(log n) | 不稳定 |
| 堆排序 | O(n log n) | O(n log n) | O(n log n) | O(1) | 不稳定 |
| 归并排序 | O(n log n) | O(n log n) | O(n log n) | O(n) | 稳定 |
| 基数排序 | O(d(n+r)) | O(d(n+r)) | O(d(n+r)) | O(n+r) | 稳定 |

## 查找算法

### 顺序查找

- **思想**：从表的一端开始，逐个比较，直到找到或查遍整个表
- **时间复杂度**：O(n)
- **适用**：无序表或有序表均可

### 二分查找

- **前提**：有序表
- **思想**：每次将查找区间对半缩小
- **时间复杂度**：O(log n)
- **适用**：顺序存储的有序表

### 分块查找（索引顺序查找）

- **思想**：将表分成若干块，块内无序，块间有序；先确定块，再在块内顺序查找
- **索引表**：存储每块的最大关键字和块的起始位置
- **时间复杂度**：O(log(n/b) + b)，b为块内元素数
- **特点**：介于顺序查找和二分查找之间

### 散列查找

- **基本思想**：根据关键码值直接访问记录
- **散列函数**：将关键字映射到存储地址
- **常见散列函数**：
  - 除留余数法：H(key) = key mod p（p取小于表长的最大素数）
  - 数字分析法：取关键字中取值较均匀的数字
  - 平方取中法：key²的中间几位
- **冲突处理方法**：
  - **开放定址法**：Hi = (H(key) + di) mod m
    - 线性探测：di = 1, 2, 3, ...
    - 平方探测：di = 1², -1², 2², -2², ...
  - **链地址法（拉链法）**：所有同义词用链表链接
  - **再散列法**：使用多个散列函数，冲突时换一个
- **装填因子** α = 记录数n / 哈希表长度m
- **平均查找长度（ASL）**：
  - 成功：取决于装填因子
  - 不成功：取决于冲突处理方法

## 算法设计与分析

### 递归算法

- **基本思想**：函数直接或间接调用自身
- **要素**：递归基（终止条件）、递归体（递推关系）
- **优缺点**：代码简洁但存在函数调用开销和栈溢出风险
- **典型算法**：二分查找、归并排序、快速排序、树的遍历、 Hanoi塔

### 贪心算法

- **基本思想**：每一步选择当前最优（局部最优）解，期望得到全局最优
- **特点**：不能保证总是得到全局最优解
- **适用**：具有贪心选择性质和最优子结构的问题
- **典型问题**：活动选择问题、哈夫曼编码、最小生成树（Prim/Kruskal）、单源最短路径（Dijkstra）

### 动态规划

- **基本思想**：将问题分解为重叠子问题，自底向上或记忆化搜索求解
- **要素**：
  - 最优子结构：全局最优解包含子问题的最优解
  - 重叠子问题：子问题被多次重复计算
  - 状态转移方程：描述子问题之间的关系
- **与贪心区别**：贪心是局部最优，动态规划考虑全局最优
- **典型问题**：最长公共子序列(LCS)、最长递增子序列(LIS)、背包问题、最短路径(Floyd)

### 分治算法

- **基本思想**：将问题分解为若干规模较小的子问题，递归求解子问题，合并子问题的解得到原问题的解
- **要素**：
  - 问题可以分解为多个相同类型的子问题
  - 子问题相互独立
  - 子问题的解可以合并为原问题的解
- **典型问题**：归并排序、快速排序、二分查找、Strassen矩阵乘法

### 算法复杂度分析

- **时间复杂度**：算法运行时间随输入规模增长的趋势
  - O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(n³) < O(2ⁿ) < O(n!)
- **空间复杂度**：算法运行过程中临时占用的存储空间
  - 包括输入空间、辅助空间、输出空间

## 考点速记

::: tip 高频考点
1. 二叉树前序+中序唯一确定二叉树；前序+后序不能唯一确定
2. 完全二叉树节点数2^k-1，深度⌈log₂(n+1)⌉
3. 哈夫曼树：n个叶子=2n-1个节点，WPL=Σ(Wi×Li)
4. 平衡二叉树平衡因子∈{-1,0,1}，四种失衡及调整方法
5. 拓扑排序：DAG的线性序列，关键路径决定工程最短工期
6. 排序稳定性：冒泡/插入/归并/基数稳定；选择/快速/堆排序不稳定
7. 快速排序平均O(n log n)，最坏O(n²)；堆排序最坏也是O(n log n)
8. 散列冲突：装填因子α越小冲突概率越低，链地址法最常用
9. 动态规划：最优子结构+重叠子问题+状态转移方程
10. 贪心算法：贪心选择性质，适用于最小生成树和Dijkstra
:::

## 重点难点

::: warning 重点难点
1. **二叉树遍历还原**：前序+中序 → 前序第一个是根 → 中序中根的位置分出左右子树 → 递归还原
2. **平衡树旋转**：LL右旋，RR左旋，LR先左后右，RL先右后左——旋转后仍满足BST性质
3. **关键路径计算**：
   - 正向求Ve（最早开始）：Ve[j] = max{Ve[i] + weight(i,j)}
   - 反向求Vl（最迟开始）：Vl[i] = min{Vl[j] - weight(i,j)}
   - 关键活动：Ve[i] == Vl[i] 的活动
4. **快速排序划分**：选基准元素，小的放左边，大的放右边，递归处理——不稳定原因：相同元素可能被交换
5. **散列函数设计**：除留余数法p取素数，避免聚集；装填因子通常取0.5~0.85
6. **递归转非递归**：用栈模拟系统调用栈，先压参数再压返回地址
:::

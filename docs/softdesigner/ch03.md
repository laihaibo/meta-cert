## 线性结构

### 数组

- **定义**：一段连续存储的相同类型数据元素的序列
- **存储方式**：顺序存储，物理地址连续
- **操作**：随机访问（O(1)）、插入（O(n)）、删除（O(n)）
- **特点**：读快写慢，空间利用率高

### 链表

- **单链表**：每个节点含数据域和指针域，只指向后继
- **双链表**：每个节点含前驱指针和后继指针
- **循环链表**：尾节点指针指向头节点，构成环
- **操作复杂度**：
  - 随机访问：O(n)
  - 插入/删除（已知位置）：O(1)
  - 插入/删除（未知位置）：O(n)（需先查找）

### 稀疏矩阵

- **定义**：矩阵中多数元素为零，仅存储非零元素
- **三元组表示法**：(行号, 列号, 值)
- **压缩存储**：节省存储空间，便于矩阵运算

---

## 栈与队列

### 栈

- **定义**：限定仅在表尾进行插入和删除的线性表（后进先出，LIFO）
- **存储方式**：顺序栈（数组）、链栈（链表）
- **基本操作**：Push（入栈）、Pop（出栈）、Peek（查看栈顶）
- **应用场景**：表达式求值、括号匹配、函数调用栈、递归转非递归

### 队列

- **定义**：限定在表头删除、表尾插入的线性表（先进先出，FIFO）
- **存储方式**：
  - **顺序队列**：数组实现，存在"假溢出"问题
  - **循环队列**：首尾相连，解决假溢出
  - **链队列**：链表实现
- **基本操作**：Enqueue（入队）、Dequeue（出队）
- **应用场景**：缓冲区、消息队列、广度优先遍历、层次遍历

### 双端队列（Deque）

- **定义**：允许在两端进行插入和删除的线性表
- **输入受限双端队列**：一端只能插入，另一端可插入删除
- **输出受限双端队列**：一端只能删除，另一端可插入删除

---

## 串

### 串的基本操作

- **定义**：由零个或多个字符组成的有限序列
- **存储方式**：定长顺序存储 / 堆分配存储 / 块链存储
- **基本操作**：赋值、连接、求子串、定位（模式匹配）

### KMP算法

- **目的**：高效的模式串匹配算法
- **核心思想**：利用已匹配部分的信息，避免不必要的回溯
- **Next数组**：记录模式串中每个位置的最长公共前后缀长度
- **时间复杂度**：O(n+m)
- **优势**：相比暴力匹配的 O(n×m) 有显著提升

---

## 数组与广义表

### 数组

- **多维数组的存储**：按行优先或按列优先存储
- **行优先（C/Java）**：行下标变化慢，列下标变化快
- **列优先（Fortran）**：列下标变化慢，行下标变化快
- **地址计算**：数组元素地址 = 基地址 + 下标偏移量×元素大小

### 广义表

- **定义**：线性表的推广，表中元素可以是原子也可以是子表
- **表示**：LS = (a₁, a₂, ..., aₙ)，其中每个aᵢ可以是原子或广义表
- **存储方式**：链式存储（头尾链表表示法）
- **深度**：广义表展开后括号嵌套的最大层数
- **长度**：最外层元素个数

## 树与二叉树（⚠️ 必考重点）

### 二叉树的基本性质

- **性质1**：第i层最多有 2^(i-1) 个节点（i≥1）
- **性质2**：深度为k的二叉树最多有 2^k - 1 个节点（k≥1）
- **性质3**：叶子节点数 n₀ = n₂ + 1（n₂为度为2的节点数）
- **性质4**：具有n个节点的完全二叉树深度 ⌈log₂(n+1)⌉
- **性质5**：完全二叉树节点编号1开始：
  - 节点i的左孩子 = 2i
  - 节点i的右孩子 = 2i+1
  - 节点i的父节点 = ⌊i/2⌋

### 二叉树遍历（⚠️ 必考还原）

- **前序遍历**（根-左-右）： NLR
- **中序遍历**（左-根-右）： LNR
- **后序遍历**（左-右-根）： LRN
- **层序遍历**：逐层从左到右
- **遍历还原规则**：
  - ✅ 已知前序+中序 → **唯一确定**二叉树
  - ✅ 已知后序+中序 → **唯一确定**二叉树
  - ❌ 已知前序+后序 → **不能唯一确定**（存在多解）

::: tip 还原口诀
前序找根（第一个）→ 中序分列（根左右）→ 左右子树递归还原
:::

### 二叉排序树（BST）

- **定义**：左子树所有节点值 < 根节点值 < 右子树所有节点值
- **中序遍历结果**：有序序列（升序）
- **查找效率**：平均 O(log n)，最坏 O(n)（退化为链表）

### 平衡二叉树（AVL树）

- **定义**：任意节点的左右子树高度差不超过1（平衡因子 ∈ {-1, 0, 1}）
- **目的**：保证操作复杂度为 O(log n)
- **失衡类型**：
  - LL型：左子树的左子树插入 → 右旋
  - RR型：右子树的右子树插入 → 左旋
  - LR型：左子树的右子树插入 → 先左旋再右旋
  - RL型：右子树的左子树插入 → 先右旋再左旋

### 哈夫曼树（⚠️ 必考计算）

- **定义**：带权路径长度（WPL）最小的二叉树
- **构建方法**：贪心算法，每次选取最小权值的两棵树合并
- **公式**：`WPL = Σ(Wi × Li)`（Wi=叶子权值, Li=路径长度）
- **哈夫曼编码**：前缀编码（任一字符编码不是另一字符编码的前缀）
- **特点**：n个叶子节点 → 共 2n-1 个节点，没有度为1的节点

::: warning 哈夫曼树计算技巧
每次取最后开始合并！权值：{2,3,5,7} → 先合并 2+3=5 → 再合并 5+5=10 → 最后合并 7+10=17 → WPL = 2×3 + 3×3 + 5×2 + 7×1 = 34
:::

### 树与森林的转换

- **森林转二叉树**：兄弟相连，父节点保留最左孩子的链接
- **二叉树转森林**：左分支恢复原树，右分支恢复原兄弟

## 图（⚠️ 遍历 + 最小生成树 + 最短路径）

### 图的存储

| 存储方式 | 空间复杂度 | 适用场景 | 查找边 |
|---------|-----------|---------|--------|
| 邻接矩阵 | O(n²) | 稠密图 | O(1) |
| 邻接表 | O(n+e) | 稀疏图 | O(e) |

### 图的遍历

| 遍历方式 | 实现 | 时间复杂度 | 应用 |
|---------|------|-----------|------|
| 深度优先 DFS | 栈/递归 | O(n+e) | 连通性、拓扑排序 |
| 广度优先 BFS | 队列 | O(n+e) | 最短路径(无权)、层次遍历 |

### 最小生成树

| 算法 | 策略 | 时间复杂度 | 适用 |
|------|------|-----------|------|
| **Prim** | 加点法（选与已选集合连接的最小边） | O(n²) | 稠密图 |
| **Kruskal** | 加边法（按权排序，不形成环） | O(e log e) | 稀疏图 |

### 最短路径

| 算法 | 类型 | 能否负权 | 时间复杂度 |
|------|------|---------|-----------|
| **Dijkstra** | 单源 | ❌ 不能 | O(n²) |
| **Floyd** | 多源 | ✅ 能(无负环) | O(n³) |
| **Bellman-Ford** | 单源 | ✅ 能，可检负环 | O(ne) |

### 拓扑排序

- **适用**：有向无环图（DAG）
- **算法**：反复选入度为0的顶点输出 → 删除其出边 → 重复
- **判定环**：输出顶点数 < 总顶点数 → 存在环

### 关键路径（⚠️ 必考计算）

- **前提**：AOE网（边=活动，顶点=事件）
- **概念**：最长路径长度 = 工程最短完成时间
- **计算公式**：
  - `Ve(j) = max{Ve[i] + weight(i,j)}`（正向，最早开始）
  - `Vl(i) = min{Vl[j] - weight(i,j)}`（反向，最迟开始）
  - `关键活动`：Ve(i) == Vl(i)

## 排序算法（⚠️ 必考对比表）

| 排序算法 | 平均时间 | 最坏时间 | 最好时间 | 空间 | 稳定性 |
|---------|---------|---------|---------|------|--------|
| 冒泡排序 | O(n²) | O(n²) | O(n) | O(1) | ✅ 稳定 |
| 直接选择 | O(n²) | O(n²) | O(n²) | O(1) | ❌ 不稳定 |
| 直接插入 | O(n²) | O(n²) | O(n) | O(1) | ✅ 稳定 |
| **快速排序** | **O(n log n)** | **O(n²)** | **O(n log n)** | **O(log n)** | ❌ 不稳定 |
| 堆排序 | O(n log n) | O(n log n) | O(n log n) | O(1) | ❌ 不稳定 |
| 归并排序 | O(n log n) | O(n log n) | O(n log n) | O(n) | ✅ 稳定 |
| 基数排序 | O(d(n+r)) | O(d(n+r)) | O(d(n+r)) | O(n+r) | ✅ 稳定 |

::: tip 排序选择口诀
"选堆归基不挑数据，快排平均最常用；需要稳定用归并，空间紧张选堆排"
:::

## 查找算法

| 查找方式 | 前提 | 时间复杂度 | 特点 |
|---------|------|-----------|------|
| 顺序查找 | 无 | O(n) | 最简单 |
| **二分查找** | **有序** | **O(log n)** | 前提有序，考最多 |
| 分块查找 | 块间有序 | O(√n) | 折中方案 |
| 散列查找 | 哈希表 | O(1) 平均 | 设计哈希函数 |

### 哈希冲突处理

- **开放定址法**：线性探测/平方探测
- **链地址法（拉链法）**：同义词链表链接 → **最常用**
- **装填因子** α = n/m，越小冲突越少

## 算法设计与分析

| 算法思想 | 核心特征 | 典型问题 |
|---------|---------|---------|
| **贪心** | 每步局部最优 | 哈夫曼、Prim/Kruskal、Dijkstra |
| **动态规划** | 最优子结构+重叠子问题+状态转移 | LCS、背包、Floyd |
| **分治** | 分解→递归→合并 | 归并排序、快排、二分 |
| **递归** | 自调用+递归基 | 二分查找、树遍历、Hanoi |

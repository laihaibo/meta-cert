## 8.1 信息安全基础

### 8.1.1 信息安全概念

**信息安全CIA三要素：**
- **保密性（Confidentiality）**：信息不被未授权访问
- **完整性（Integrity）**：信息不被未授权篡改
- **可用性（Availability）**：授权用户能正常访问

**信息安全属性扩展：**
- **真实性**：信息来源真实
- **不可否认性**：发送方不能否认发送行为
- **可控性**：对信息的传播和内容有控制能力
- **可审计性**：行为可追溯

### 8.1.2 信息安全威胁

**威胁类型：**
- **截获（Interception）**：窃听、流量分析
- **中断（Interruption）**：破坏、拒绝服务
- **篡改（Modification）**：数据篡改
- **伪造（Fabrication）**：身份伪造、数据伪造

**攻击类型：**
- **被动攻击**：窃听、流量分析（难以检测，注重预防）
- **主动攻击**：中断、篡改、伪造、重放（注重检测）

### 8.1.3 信息安全体系

**OSI安全体系结构：**
- **安全服务：**
  - 认证服务（对等实体认证、数据源认证）
  - 访问控制服务
  - 数据保密性服务
  - 数据完整性服务
  - 不可否认服务
- **安全机制：**
  - 加密机制
  - 数字签名机制
  - 访问控制机制
  - 数据完整性机制
  - 认证交换机制
  - 流量填充机制
  - 路由控制机制
  - 公证机制

## 8.2 加密技术

### 8.2.1 密码学基础

**密码体制分类：**
- **对称加密（私钥加密）：** 加密密钥 = 解密密钥
- **非对称加密（公钥加密）：** 加密密钥 ≠ 解密密钥
  - 公钥公开，私钥保密
  - 加密用公钥，解密用私钥（保密通信）
  - 签名用私钥，验证用公钥（数字签名）

### 8.2.2 对称加密算法

| 算法 | 密钥长度 | 特点 |
|------|---------|------|
| DES | 56位 | 已不安全，被AES取代 |
| 3DES | 112/168位 | DES的三重应用，安全性提升 |
| AES | 128/192/256位 | 现行标准，速度快 |
| IDEA | 128位 | PGP中使用 |
| RC4 | 可变 | 流密码，WEP/TLS曾用 |

**对称加密特点：**
- 优点：加密速度快，适合大量数据
- 缺点：密钥分发困难，无法验证身份

### 8.2.3 非对称加密算法

| 算法 | 密钥长度 | 特点 |
|------|---------|------|
| RSA | 可变（常用1024/2048位） | 最广泛，基于大数分解 |
| ECC | 160位（等效于RSA 1024位） | 椭圆曲线，密钥短效率高 |
| Diffie-Hellman | 可变 | 密钥交换协议 |
| ElGamal | 可变 | 基于离散对数 |
| DSA | 1024/2048位 | 数字签名专用 |

**非对称加密特点：**
- 优点：密钥管理方便，支持数字签名
- 缺点：加密速度慢，不适合大量数据

### 8.2.4 混合加密系统

**实际应用采用混合加密：**
1. 用非对称加密传输对称密钥
2. 用对称加密传输实际数据

**过程：**
```
发送方：
  生成会话密钥 → 用接收方公钥加密会话密钥 → 用会话密钥加密数据 → 发送
接收方：
  用私钥解密会话密钥 → 用会话密钥解密数据
```

### 8.2.5 消息摘要（哈希函数）

**哈希函数特性：**
- 单向性：从消息难以反推哈希值
- 抗碰撞性：难以找到两个消息有相同哈希值
- 确定性：同一消息产生相同哈希值

**常用哈希算法：**

| 算法 | 输出长度 | 状态 |
|------|---------|------|
| MD5 | 128位 | 已不安全 |
| SHA-1 | 160位 | 已不安全 |
| SHA-256 | 256位 | 安全标准 |
| SHA-3 | 可变 | 最新标准 |

**应用：**
- 验证数据完整性
- 数字签名的预处理
- 密码存储（加盐）

## 8.3 数字签名与认证

### 8.3.1 数字签名

**目的：** 验证身份、保证完整性、不可否认

**过程：**
1. 发送方用私钥对消息摘要进行加密（签名）
2. 发送消息 + 签名给接收方
3. 接收方用公钥解密签名得到摘要A
4. 接收方对消息计算摘要B
5. 比较A和B，验证签名

**特点：**
- 只有私钥持有者能生成有效签名
- 任何人可用公钥验证签名
- 消息任何改动都会导致验证失败

### 8.3.2 数字证书

**数字证书内容：**
- 证书版本
- 证书序列号
- 签名算法标识
- 颁发者（CA）
- 有效期
- 主体（证书持有者）
- 主体公钥信息
- 颁发者数字签名

**证书颁发机构（CA）：**
- 负责签发和管理数字证书
- 是信任链的根节点
- 常用CA：VeriSign、GeoTrust、Let's Encrypt

**PKI（公钥基础设施）：**
- 提供公钥加密和数字签名服务
- 核心：CA、证书、CRL（证书撤销列表）
- 使用数字证书来绑定公钥和身份

### 8.3.3 认证协议

**Kerberos：**
- 网络认证协议，使用对称加密
- 三方：客户端、服务器、KDC（密钥分发中心）
- 使用票据（Ticket）机制
- 特点：单点登录，支持分布式环境

**X.509：**
- 公钥证书标准格式
- 用于SSL/TLS、S/MIME、IPsec等

## 8.4 网络安全技术

### 8.4.1 防火墙

**防火墙定义：**
位于内部网络和外部网络之间的安全屏障，根据安全策略控制进出网络的流量。

**防火墙分类：**

| 类型 | 工作层次 | 特点 |
|------|---------|------|
| 包过滤防火墙 | 网络层 | 基于IP、端口过滤，速度快 |
| 状态检测防火墙 | 传输层 | 跟踪连接状态，安全性更高 |
| 应用层代理防火墙 | 应用层 | 检查应用层数据，最安全但最慢 |
| 电路级网关 | 会话层 | 验证TCP/UDP连接 |

**防火墙功能：**
- 访问控制：根据规则过滤数据包
- 网络地址转换（NAT）：隐藏内部IP
- 日志记录：记录连接尝试
- 隔离：保护内部网络

### 8.4.2 入侵检测系统（IDS）

**IDS分类：**

| 类型 | 描述 | 优点 | 缺点 |
|------|------|------|------|
| 基于特征（误用检测） | 匹配已知攻击模式 | 误报率低 | 无法检测未知攻击 |
| 基于异常（异常检测） | 检测偏离正常行为 | 可检测未知攻击 | 误报率高 |

**部署位置：**
- 基于网络（NIDS）：监控网络流量
- 基于主机（HIDS）：监控主机日志和行为

### 8.4.3 VPN（虚拟专用网络）

**VPN技术：**
- 在公共网络上建立加密隧道
- 实现远程安全访问
- 常用协议：
  - PPTP：点对点隧道协议
  - L2TP：第二层隧道协议
  - IPSec：网络层安全协议
  - SSL/TLS VPN：应用层VPN
  - OpenVPN：开源VPN解决方案

### 8.4.4 网络攻击与防护

**常见攻击：**

| 攻击类型 | 描述 | 防护措施 |
|---------|------|---------|
| 拒绝服务（DoS） | 耗尽目标资源 | 防火墙、流量清洗 |
| 分布式拒绝服务（DDoS） | 多源拒绝服务 | CDN、DDoS防护 |
| 缓冲区溢出 | 利用程序漏洞 | 输入验证、内存保护 |
| SQL注入 | 注入恶意SQL | 参数化查询、输入过滤 |
| XSS（跨站脚本） | 注入恶意脚本 | 输入过滤、输出编码 |
| CSRF（跨站请求伪造） | 伪造用户请求 | 令牌验证、Referer检查 |
| 中间人攻击 | 拦截通信 | TLS加密、证书验证 |
| 钓鱼攻击 | 伪装可信网站 | 用户教育、DMARC |

## 8.5 应用安全

### 8.5.1 操作系统安全

**用户认证：**
- 用户名+密码
- 双因素认证（2FA）：密码 + 短信/硬件令牌
- 生物特征：指纹、虹膜、面部识别
- 智能卡：物理令牌

**访问控制：**
- **自主访问控制（DAC）**：资源所有者决定访问权限
- **强制访问控制（MAC）**：系统管理员统一管理，基于安全级别
- **基于角色访问控制（RBAC）**：基于用户角色授权

### 8.5.2 恶意软件

**恶意软件类型：**
- **病毒**：寄生在可执行文件中
- **蠕虫**：独立传播，利用网络漏洞
- **特洛伊木马**：伪装成合法软件
- **后门**：绕过正常认证机制
- **逻辑炸弹**：特定条件触发
- **僵尸网络**：被控制的计算机网络
- **勒索软件**：加密文件索要赎金

**防护措施：**
- 防病毒软件
- 及时更新系统补丁
- 用户安全意识和培训
- 数据备份

### 8.5.3 数据库安全

**安全特性：**
- **数据加密**：存储加密、传输加密
- **访问控制**：用户权限管理
- **审计跟踪**：记录操作日志
- **完整性约束**：保证数据一致性
- **备份恢复**：定期备份，灾难恢复

## 8.6 法律法规与道德

### 8.6.1 信息安全法律法规

**《网络安全法》要点：**
- 网络运营者安全保护义务
- 关键信息基础设施保护
- 数据跨境传输规定
- 违法行为的法律责任

**《数据安全法》要点：**
- 数据分类分级保护
- 数据安全审查
- 数据处理者义务

**《个人信息保护法》要点：**
- 个人信息处理需同意
- 敏感个人信息特殊保护
- 个人权利（查阅、更正、删除等）
- 跨境传输规则

### 8.6.2 职业道德

- **保密义务**：保护用户数据和商业秘密
- **专业能力**：持续学习，保持专业水平
- **社会责任**：不制作、传播恶意软件
- **合规操作**：遵守法律法规

::: warning 公式
- RSA密钥生成：选择p、q，n=p×q，φ(n)=(p-1)(q-1)，选择e与φ(n)互质，计算d≡e^(-1) mod φ(n)
- 加密：C = M^e mod n
- 解密：M = C^d mod n
- 数字签名：S = H(M)^d mod n，验证：H(M) = S^e mod n

## 考情分析

本章是软件设计师考试的重要章节，主要介绍操作系统的基本概念、进程管理、内存管理、文件系统等。考试重点在于进程调度算法、内存管理方式和死锁处理。

## 4.1 操作系统概述

### 4.1.1 操作系统基本概念

**操作系统定义：**
操作系统是计算机系统的核心软件，它管理和控制计算机硬件与软件资源，合理地组织计算机工作流程，并为用户提供良好的人机界面。

**操作系统功能：**
- **处理机管理**：进程调度、同步、通信
- **存储器管理**：内存分配、地址映射、保护
- **设备管理**：设备分配、缓冲区管理、I/O控制
- **文件管理**：文件存储、访问控制、目录管理
- **用户接口**：命令接口、程序接口（系统调用）

### 4.1.2 操作系统分类

**按用户数分类：**
- **单用户操作系统**：MS-DOS
- **多用户操作系统**：UNIX、Linux

**按任务数分类：**
- **单任务操作系统**：一次只执行一个任务
- **多任务操作系统**：可同时执行多个任务

**按体系结构分类：**
- **网络操作系统**：支持网络通信和资源共享
- **分布式操作系统**：多台计算机协同工作
- **实时操作系统**：必须在规定时间内完成处理

### 4.1.3 系统调用

**系统调用是用户程序与操作系统之间的接口。**

**常用系统调用：**
- 进程控制：创建进程、终止进程、等待进程
- 文件操作：创建文件、打开文件、读/写文件、关闭文件
- 设备管理：设备请求、设备释放
- 信息维护：获取系统时间、进程信息等

## 4.2 进程管理

### 4.2.1 进程与线程

**进程：**
- 程序的一次执行过程
- 是系统进行资源分配和调度的基本单位
- **进程状态：** 运行、就绪、阻塞（等待）

**进程控制块（PCB）：**
- 操作系统用于管理进程的数据结构
- 包含：进程标识符、进程状态、CPU现场保护区、调度信息、资源清单等

**线程：**
- 进程内的一个执行单元
- 是CPU调度的基本单位
- **优点：** 减少并发执行时的开销，提高并发性

**进程与线程比较：**

| 比较项 | 进程 | 线程 |
|--------|------|------|
| 资源拥有 | 拥有资源的基本单位 | 不拥有资源，可共享所属进程的资源 |
| 调度 | 进程是调度基本单位 | 线程是CPU调度基本单位 |
| 并发性 | 不同进程可以并发执行 | 同一进程内线程可以并发执行 |
| 开销 | 创建、撤销开销大 | 创建、撤销开销小 |
| 通信 | 需要IPC机制 | 可直接读写进程数据段进行通信 |

### 4.2.2 进程调度

**调度层次：**
- **高级调度（作业调度）**：决定哪些作业可以进入内存
- **中级调度（交换调度）**：决定哪些进程驻留内存（用于虚拟存储）
- **低级调度（进程调度）**：决定就绪队列中哪个进程获得CPU

**调度算法：**

| 算法 | 描述 | 优点 | 缺点 |
|------|------|------|------|
| FCFS（先来先服务） | 按到达顺序排队 | 公平、简单 | 平均等待时间长，不利于短作业 |
| SJF（最短作业优先） | 选择预计运行时间最短的作业 | 平均等待时间短 | 可能产生饥饿现象 |
| SRT（最短剩余时间优先） | 抢占式SJF | 响应快 | 频繁切换，开销大 |
| RR（时间片轮转） | 每个进程分配固定时间片 | 公平，响应快 | 时间片大小影响效率 |
| HPF（最高优先级） | 选择优先级最高的进程 | 适合实时系统 | 低优先级进程可能饥饿 |
| MFQ（多级反馈队列） | 多个队列，不同优先级 | 兼顾效率和响应 | 参数调整复杂 |

### 4.2.3 进程同步与通信

**临界资源和临界区：**
- 临界资源：一次只允许一个进程访问的资源
- 临界区：进程中访问临界资源的代码段

**同步机制应满足：**
1. **空闲让进**：无进程在临界区时，允许进程进入
2. **忙则等待**：有进程在临界区时，其他进程等待
3. **有限等待**：等待进入临界区的进程应能在有限时间内进入
4. **让权等待**：进程不能进入临界区时，应释放CPU

**信号量机制：**
- `P操作`（wait）：信号量减1，若为负则阻塞
- `V操作`（signal）：信号量加1，若小于等于0则唤醒一个等待进程

**PV操作解决经典问题：**
- 生产者-消费者问题
- 读者-写者问题
- 哲学家进餐问题

**进程通信方式：**
- **共享存储区**：进程间共享内存区域
- **消息传递**：发送/接收消息（同步/异步）
- **管道通信**：半双工通信，父子进程间

### 4.2.4 死锁

**死锁产生的四个必要条件：**
1. **互斥条件**：资源一次只能被一个进程占用
2. **请求和保持条件**：进程已占有资源，又请求新资源
3. **不可剥夺条件**：已分配资源不能被强行剥夺
4. **环路等待条件**：存在进程-资源的环形链

**死锁预防：**
- 破坏互斥条件：使资源可共享（不实际）
- 破坏请求和保持：一次性申请所有资源
- 破坏不可剥夺：允许剥夺资源（如抢占）
- 破坏环路等待：资源有序分配法

**死锁避免：**
- **银行家算法**：分配前检查是否安全

**死锁检测与恢复：**
- 检测：利用资源分配图简化法
- 恢复：剥夺资源、撤销进程

## 4.3 内存管理

### 4.3.1 存储管理方式

**连续分配方式：**
- **单一连续分配**：内存分为系统区和用户区
- **固定分区分配**：内存分为若干个固定大小的分区
- **可变分区分配**：根据进程大小动态划分
  - 分配算法：首次适应、最佳适应、最坏适应

**非连续分配方式（离散分配）：**
- **分页存储管理**：将进程和内存分成固定大小的页/块
  - 页号 + 页内偏移 = 物理地址
  - 页表：记录页号到物理块号的映射
- **分段存储管理**：按逻辑意义分段（代码段、数据段、堆栈段）
  - 段号 + 段内偏移 = 物理地址
  - 段表：记录段基址和段长
- **段页式存储管理**：先分段，每段再分页

### 4.3.2 虚拟存储

**虚拟存储实现基础：**
- 程序执行的局部性原理（时间局部性、空间局部性）
- 内存驻留只需当前执行的部分程序

**页面置换算法：**
- **OPT（最佳置换）**：淘汰最久不使用的页面（理论最优，无法实现）
- **FIFO（先进先出）**：淘汰最早进入的页面
- **LRU（最近最少使用）**：淘汰最近最少使用的页面
- **LFU（最少使用）**：淘汰访问次数最少的页面

### 4.3.3 页面置换算法比较

| 算法 | 实现 | 优点 | 缺点 |
|------|------|------|------|
| OPT | 无法实现（需预知未来） | 最优 | 仅用于评价 |
| FIFO | 简单 | 实现简单 | Belady异常（分配页数增加缺页率升高） |
| LRU | 需硬件支持或软件模拟 | 较好效果 | 实现较复杂 |
| LFU | 计数访问次数 | 适应访问频率变化 | 不反映时间局部性 |

### 4.3.4 页面分配策略

- **固定分配**：为每个进程分配固定数量的页框
- **可变分配**：根据进程需求动态调整
- **全局置换**：可从其他进程选页置换
- **局部置换**：只能从自身选页置换

## 4.4 文件系统

### 4.4.1 文件基本概念

**文件定义：** 具有符号名的一组相关数据的有序序列

**文件类型：**
- 按性质：系统文件、用户文件、库文件
- 按用途：源文件、目标文件、可执行文件
- 按存取方式：顺序文件、随机文件
- 按组织形式：流式文件、记录文件

**文件属性：**
- 文件名、类型、物理位置、大小、创建时间、访问权限等

**文件操作：**
- 创建、删除、打开、关闭、读、写、定位、截断

### 4.4.2 文件目录

**目录结构类型：**
- **单级目录**：所有文件在一个目录中（简单但易重名）
- **两级目录**：主目录 + 用户文件目录
- **树形目录**：层次结构（现代OS常用）
- **无环图目录**：允许文件共享的树形目录

**路径名：**
- **绝对路径**：从根目录开始的完整路径
- **相对路径**：从当前目录开始的路径

### 4.4.3 文件存储空间管理

**空闲空间管理方法：**
- **空闲文件表**：记录空闲文件
- **空闲链表**：空闲块链接成链表
- **位示图**：用二进制位表示块的使用情况
- **成组链接法**：UNIX系统采用

### 4.4.4 磁盘调度

**磁盘访问时间组成：**
- 寻道时间：磁头移动到目标磁道（最大开销）
- 旋转延迟时间：等待目标扇区到达磁头
- 传输时间：数据读写时间

**磁盘调度算法：**

| 算法 | 描述 | 优点 | 缺点 |
|------|------|------|------|
| FCFS | 按请求顺序 | 公平、简单 | 寻道时间可能很长 |
| SSTF（最短寻道优先） | 选择离磁头最近的请求 | 寻道时间短 | 可能产生饥饿 |
| SCAN（电梯算法） | 磁头单向移动，处理所有请求 | 寻道时间较短 | 对两端请求不公平 |
| C-SCAN（循环扫描） | 磁头单向扫描，到尽头后返回 | 响应时间均匀 | 寻道时间略长 |
| LOOK | SCAN改进，到最后一个请求后反向 | 减少空移动 | |
| C-LOOK | C-SCAN改进 | 同上 | |

::: tip 高频考点
1. 进程状态转换（运行→就绪：时间片到；运行→阻塞：I/O请求；就绪→运行：被调度）
2. 调度算法比较：FCFS、SJF、RR、HPF的优缺点
3. 死锁产生的四个必要条件及破坏方法
4. 银行家算法（安全性检查）
5. 页表的作用和地址变换过程
6. 页面置换算法：OPT、FIFO、LRU
7. Belady异常：FIFO算法分配页数增加时缺页率反而升高
8. 磁盘调度算法：SSTF、SCAN、C-SCAN
9. 文件目录结构：单级、两级、树形
:::

::: warning 计算公式
- CPU利用率 = 1 - 缺页率
- 缺页中断率 = 缺页次数 / 总访问次数
- 磁盘访问时间 = 寻道时间 + 旋转延迟 + 传输时间
- 寻道时间公式：T = m × n + b（m：移动时间常数，n：移动道数，b：启动时间）
:::
